## 1. エグゼクティブサマリー

### 全国対応できるか？

**結論: 現状では不可能。**

LINE Botは東京・神奈川・千葉・埼玉の4都県のみ実質対応しており、残り43道府県のユーザーには**対象外であることを一切告知せず、関東の求人を提示する**という致命的な設計になっている。400件のシミュレーションのうち**63.2%（253件）がGEO_LOCK（地域ロック）**に該当し、Criticalが119件、Highが157件。全国展開どころか、現状の「その他の地域」選択肢の存在自体がユーザー体験を破壊している。

### 主要ボトルネック トップ3

1. **GEO_LOCK（地域ロック）**: 全400件の63.2%。「その他の地域」を選んだユーザーに対象外であることを一度も伝えず、関東の求人を表示して終わる。ユーザーは「なぜ関東？」と混乱し、サービスへの信頼を完全に失う。
2. **INPUT_LOCK（自由テキスト無視）**: 「北海道」「名古屋で探したい」等の自由テキスト入力を一切パースせず、ボタン再表示を繰り返す。LINEはチャットUIなのにテキスト入力を無視するのは致命的。
3. **HUMAN_HANDOFF_FAIL（人間引継ぎ不在）**: 「限界」「辞めたい」等の緊急性の高い発言や、エリア外で怒っているユーザーへの即時人間引き継ぎが一切ない。

## 3. 県別評価

### 対応済み4県（東京/神奈川/千葉/埼玉）の品質

**56件中、NONE（正常動作）30件 = 53.6%**

| 指標 | 結果 |
|------|------|
| 正常動作率 | 53.6%（30/56件） |
| JOB_MATCH_FAIL | 12件（21.4%） |
| UX_DROP/OTHER | 7件（12.5%） |
| INPUT_LOCK | 3件 |
| Low/None severity | 38件（67.9%） |
| Critical/High | 4件（7.1%） |

**評価:** 対応エリア内では概ね正常に動作する。主な問題はJOB_MATCH_FAIL（ICU/NICU/美容/透析等の専門分野がサブカテゴリにない）と、自由テキスト入力時のINPUT_LOCK。致命的な問題は少ないが、改善余地はある。

### 非対応43道府県の共通問題

**344件中、GEO_LOCKを含む件数: 252件（73.3%）**

共通パターン:
1. 「その他の地域」をタップ → 対象外であることの告知なし
2. 全質問（施設種別/診療科/働き方/緊急度）に回答させる
3. マッチング結果として関東の施設を表示
4. ユーザー混乱・離脱

**これは「詐欺的」と言われても仕方がないUX。** 質問に全て答えさせた後に全く関係ない地域の求人を出すのは、ユーザーの時間を奪い、期待を裏切る行為。

### 地域特有の問題

#### 静岡県（10件）
- 県東部（熱海・三島・沼津）は神奈川通勤圏だが、現行Bot上で判別不可
- 「その他の地域」を選んだ後に神奈川の求人が出ると「近い！」と思う人もいるが、意図的な提案ではないためミスマッチ

#### 山梨（5件）
- 東京への新幹線通勤者が実在するため、「東京」を選べばBotは正常動作
- ただしil_areaの質問が「お住まいの地域」なら山梨在住者は「その他」を選ぶ → GEO_LOCK

#### 大阪（29件）— **エリア拡大最優先候補**
- 25件がGEO_LOCK関連。看護師人口は東京に次ぐ規模
- 「関西圏の求人はいつ対応しますか？」という質問が想定される
- エリア拡大時の第1候補として経営判断が必要

#### 愛知（17件）
- 名古屋圏750万人。15件がGEO_LOCK
- 東海地方の中核であり、愛知対応=岐阜・三重もカバー可能

#### 福岡（12件）
- 看護師養成校が多く、九州全域の玄関口
- 8件がGEO_LOCK。九州展開のハブとして重要

#### 広島（14件）
- 中国地方の中核都市。10件がGEO_LOCK
- 広島対応=岡山・山口への足がかり

## 5. 再設計提案

### 5.1 「その他の地域」選択時の正直な説明文

**現状:**
> 「その他の地域ですね！候補: 24,488件」→ 施設種別の質問へ

**改善案:**
> 「現在、ナースロビーでは**東京・神奈川・千葉・埼玉**の求人をご紹介しています。
> 
> お住まいの地域は準備中ですが、以下からお選びください:
> 
> [関東の求人を見てみる] [対応エリア拡大時に通知を受け取る] [スタッフに直接相談する]」

ポイント:
- 対象外であることを**最初に**伝える
- 選択肢を3つ用意し、ユーザーの意思で次のアクションを決めさせる
- 「通知を受け取る」でリードを回収する
- 「直接相談する」で人間引き継ぎの選択肢を残す

### 5.2 全国向け導線改善

**対応エリア外ユーザーへの誠実な対応フロー:**

```
[ユーザー] 友だち追加
    ↓
[Bot] Welcome + 対応エリア明示（東京/神奈川/千葉/埼玉）
    ↓
[Bot] il_area表示（4県 + その他の地域）
    ↓
[ユーザー] 「その他の地域」タップ
    ↓
[Bot] 正直な説明 + 3選択肢
    ↓
  ├→ [関東を見る] → 通常フロー
  ├→ [通知希望] → LINE ID保存 + 「ありがとうございます」 + 転職お役立ち情報配信
  └→ [相談する] → Slack通知 → 人間対応
```

### 5.3 初期質問フロー最適化

**現状の問題:** 全5ステップ（エリア→施設→診療科→働き方→緊急度）を完走させてから結果を出す。エリア外ユーザーは5ステップ全てが無駄になる。

**改善案:**
1. **il_area（エリア選択）を最初のゲート**にする — 現状通り
2. エリア外なら**即座に**分岐 — 5ステップ完走させない
3. エリア内なら残り4ステップへ
4. 各ステップで「わからない/決めてない」の選択肢を追加

### 5.4 LINE離脱抑制策

| 離脱タイミング | 原因 | 対策 |
|--------------|------|------|
| il_area直後 | エリア外と気づいた | 上記5.2のフロー |
| 質問途中 | 選択肢にない条件 | 「その他」ボタン + フリーテキスト |
| matching_preview後 | 求人が合わない | 「条件を変える」+「スタッフに相談」 |
| 結果0件 | 該当なし | 隣接エリア提案 + 条件緩和提案 |
| 自由テキスト無視後 | Bot感への失望 | NLP + 3回ループ→人間引き継ぎ |

### 5.5 人への引継ぎ基準の見直し

**現状:** 引継ぎ基準が不明確。Botがスタックしても人間に繋がない。

**改善案 — 即時引き継ぎトリガー:**
1. ネガティブキーワード検出（「限界」「辞めたい」「パワハラ」「被災」等）
2. エリア外 + 「すぐ転職したい」
3. 同一テキスト3回以上入力（ループ検出）
4. 怒り/不満の感情表現検出
5. 離島・へき地からの相談
6. 管理職（師長/主任）経験者の特殊条件
7. 新卒内定辞退等の緊急ケース

**引き継ぎ方法:** Slack通知（#ロビー小田原人材紹介）+ ユーザーへ「担当スタッフにお繋ぎします。少々お待ちください」表示

### 5.6 自由テキスト入力の活用

**現状:** 自由テキスト入力は全て「もう一度お選びください」で拒否。

**改善案（段階的）:**

**Phase 1（即時）:** 都道府県名/市区町村名のキーワードマッチング
- 「北海道」→ エリア外告知
- 「横浜」→ 神奈川にマッピング
- 「名古屋」→ エリア外告知

**Phase 2（1週間）:** 施設種別/診療科のキーワード抽出
- 「ICU」→ 急性期病院+ICU系でマッチング
- 「日勤のみ」→ 働き方にマッピング

**Phase 3（1ヶ月）:** 文脈理解（OpenAI API活用）
- 「夜勤がつらくて日勤に変えたい」→ 働き方=日勤のみ
- 「もう辞めたいんですけど」→ 人間引き継ぎ

### 5.7 広告ターゲティングとの整合性

**現状の問題:** SNS広告が全国配信されており、非対応エリアのユーザーを集客してしまっている。

**改善案:**
1. **Meta広告:** ジオターゲティングを関東4都県+近隣県に限定（即時実施可）
2. **TikTok:** 同上。加えて、動画内に「関東エリア限定」テロップを追加
3. **Instagram:** プロフィールに「東京・神奈川・千葉・埼玉の看護師転職」を明記
4. **SEO:** area/guide配下の既存コンテンツは全国対応で問題ないが、LP-Aに「対応エリア: 関東」を追記
5. **Google広告（将来）:** 配信地域を厳密に設定

**効果:** 非対応エリアへの広告費を0にすることで、CPA（獲得単価）が大幅改善される。現状、広告で集めたユーザーの相当数がエリア外で離脱しており、この広告費は完全な無駄。

## 結論

**LINE Botの現状は「関東専用Bot」であり、「全国対応」とは程遠い。** しかし、修正項目の多くは大規模な再設計を必要としない。特にCritical項目のC1（「その他の地域」選択時の正直な説明文）とC2（広告ジオターゲティング）は、数時間の作業で実装可能であり、ユーザー体験とCPAを劇的に改善できる。

エリア拡大は経営判断だが、**拡大前に今すぐやるべきは「嘘をつかないこと」**。対象外エリアのユーザーに対して誠実に「現在は関東のみ」と伝え、通知希望を受け付けるだけで、ブランド毀損を防ぎつつ将来の顧客リストを構築できる。
