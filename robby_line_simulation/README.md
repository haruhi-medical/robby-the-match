# ロビーLINE Bot 全国シミュレーション検証

## 目的
ナースロビーのLINE Bot導線が全国47都道府県の求職者に対して自然に機能するかを検証する。

## 件数
- 実行エージェント: 8名
- 各エージェント: 50ケース
- 合計: **400ケース**
- 全国47都道府県を必ずカバー（各県最低4件）

## 検証対象システム概要
- LINE Bot: Cloudflare Worker（worker.js）
- DB: D1（24,488施設 — 東京/神奈川/千葉/埼玉のみ）
- 会話フロー: il_area → il_subarea → il_facility_type → il_department → il_workstyle → il_urgency → matching_preview
- エリア選択: 東京/神奈川/千葉/埼玉/その他の地域
- 「その他」→ undecided → 関東全域の求人にフォールバック（北海道〜沖縄の求人なし）

## 既知の構造的課題（コード分析時点）
1. **GEO_LOCK**: D1 DBに東京/神奈川/千葉/埼玉の施設しかない。全国対応不可。
2. **「その他の地域」の挙動**: other → undecided_il → 横浜・川崎・23区・多摩・さいたま・千葉にマッピング。地方ユーザーに関東の求人を出す。
3. **候補数表示の誤解**: 「その他の地域」選択時に全24,488件が候補として表示されるが、実際にはマッチング時にエリアフィルタが効かず全件から抽出。
4. **エリア選択UIに県名なし**: 地方ユーザーは自県が選べない。
