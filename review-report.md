# ナースロビー 外部レビューレポート

> 作成日: 2026-04-14
> レビュアー: 外部コンサルタント（医療人材紹介ビジネス）
> 対象: プロジェクト全体（コード・コンテンツ・戦略・運用）

---

## 1. エグゼクティブサマリ

**技術基盤は異常なほど充実しているが、売上はゼロ。LINE登録もゼロ。PV/日は3。** 7週間で87ページのSEOコンテンツ、24,488施設DB、2,936求人DB、25本のcronジョブ、LINE Botの20機能を構築したが、それら全てが「看護師1人もLINE登録していない」という現実の前では空回りしている。Mac Mini 1台に事業の全インフラが依存しており、ビジネス継続性の観点で危険。

---

## 2. 致命的な問題（事業継続に関わるもの）

### 2-1. 売上ゼロ・リード（LINE登録）ゼロが7週間継続

STATE.mdのKPIが全てを物語っている:
- LINE登録数: **0**（目標: Month2で5）
- 成約数: **0**（目標: Month3で1）
- PV/日: **3**（目標: 100）
- インデックス数: **17/87**（80%がGoogleに認識されていない）

87ページ書いても3PV/日ということは、誰も来ていない。25本のcronが毎日回っても、そこに看護師はいない。投稿キュー61件が「ready」でも、それを見る人がいない。**全リソースの95%が「ファネルの入口に人が来る前のシステム構築」に費やされている。**

### 2-2. 単一障害点: Mac Mini M4

事業の全インフラが1台のMac Miniに依存している:
- 25本のcronジョブ（SEO、コンテンツ生成、SNS投稿、監視、レポーティング）
- LINE Bot応答のCloudflare Worker以外、全てこのマシンから実行
- Slack連携、Git push、Workerデプロイも全てここ
- **マシンが壊れたら、復旧に何日かかるか見積もりすらない**
- スリープ無効化は対応済みだが、それ以外の物理障害リスクは残る

### 2-3. Meta広告が「出稿待ち」のまま止まっている

STATE.mdの優先度Aに「Meta広告出稿（¥2,000/日）」が書かれているが、v6は¥6,108消化で0登録、v7は「社長がAds Managerで設定」待ちのまま。日予算¥2,000で月¥60,000。仮にCPA ¥5,000で12件/月のLINE登録を獲得できても、そこからヒアリング→面談→内定→入職のコンバージョンを掛けると、月1件の成約に到達するのは数ヶ月後。**広告が動いていないのに、広告レポートのcronが毎朝動いている。**

### 2-4. 紹介実績ゼロのまま訴求している

許可番号（23-ユ-302928）取得済み、病院側の受け入れ体制も確保済み。あとは看護師を集めるだけ。しかし:
- **紹介実績ゼロ**の状態で「手数料10%」を訴求しても、看護師からの信頼が得にくい
- 「元病院人事」とLPに書いているが、具体的な経験・実績の記載がない
- **最初の1件の成約実績を作ることが、サイト全体の信頼性を劇的に変える**

## 3. 重要な改善点（3ヶ月以内にやるべき）

### 3-1. 広告を今日出す

技術の完成度を上げることと、看護師を1人獲得することは別の仕事。¥2,000/日の広告は「社長が設定するのを待つ」ものではなく、事業の生命線。v6で失敗した原因分析（Audience Network配信、LINE直リンク不通）は済んでいるのだから、v7を今すぐ出すべき。

### 3-2. SEOインデックスの修正

87ページ中17ページしかインデックスされていない。80%がGoogleに無視されている状態。原因として考えられるのは:
- sitemapのlastmodが全て同一日（2026-04-01に一括更新した形跡）
- コンテンツの類似度が高い（AI生成の地域別ページが同じテンプレートで量産されている）
- ドメイン年齢が2ヶ月程度で、まだ信頼が蓄積されていない
- 内部リンクの構造が浅い（各ページからトップLPへのリンクはあるが、記事間の文脈的リンクが弱い）

### 3-3. コンテンツの「E-E-A-T」強化

10ページをサンプリングした所見:
- **Experience（経験）**: 「元病院人事」以上の具体性がない。「○年間で○人の看護師を紹介した」等の実績がゼロ
- **Expertise（専門性）**: FAQ構造化データは充実。ただし回答が教科書的で、現場感がない
- **Authoritativeness（権威性）**: 著者が「神奈川ナース転職編集部」という匿名組織。Googleはこれを信頼しない
- **Trustworthiness（信頼性）**: 許可番号の記載あり。ただしプライバシーポリシーがnoindexなので、Googleからは見えない

**特に問題なのは、ブログと地域ページのコンテンツ品質。** AI生成感が強く、「小田原市の看護師求人・転職情報を網羅」と言いながら具体的な求人件数や病院名の情報が構造化データのFAQ部分にしか存在しない。本文にリアルな数字が入っていない。

### 3-4. cron25本の費用対効果の棚卸し

以下のcronは現時点で成果に貢献していない:
- `pdca_competitor.sh`（毎日10:00）: 競合分析だが、分析結果がアクションに変換されている形跡がない
- `cron_autoresearch.sh`（毎日02:00）: コスト全体の98%を占めるが、生成コンテンツを見る人がいない段階で品質改善しても意味がない
- `pdca_weekly_content.sh`（日曜05:00）: Claude CLI認証エラー（exit_code:78）で失敗し続けている
- `instagram_engage.py`（毎日12:00）: 非公式APIでのエンゲージメント自動化はBAN直結リスク（STATE.mdにも「BAN/CAPTCHAで停止。使うな」と書いてある旧方式と同じライブラリ instagrapi を使用）

### 3-5. 病院側の営業活動の着手

**病院側の受け入れ体制は既にある。看護師さえ集まればすぐ紹介できる状態。** ボトルネックは純粋に看護師の集客のみ。であれば尚更、広告・SEO・SNSの「人を連れてくる」部分に全リソースを集中すべき。25本のcronのうち、集客に直接貢献しているジョブがどれだけあるか棚卸しが必要。

## 4. 中期施策（6ヶ月スパン）

### 4-1. 実績の獲得と公開

最初の1件の成約を最優先で獲得し、その体験談（匿名可）をサイトに掲載する。「看護師の声」がゼロのサイトは、どれだけSEOを頑張っても信頼されない。

### 4-2. LP→LINE→成約のファネル最適化

LINE Botの20機能は多すぎる。現時点で必要なのは:
1. 友だち追加
2. 3問のクイックヒアリング
3. 求人カード表示
4. 担当者引き継ぎ（Slack通知）

アキネーター型UI、AI相談、NLP解析、診療科フィルタ、条件変更UI、ナーチャリングステート等は、月100件のLINE登録が来てから考えるべき。

### 4-3. Googleビジネスプロフィールの登録

「後回し」リストに入っているが、ローカルSEOの生命線。「神奈川 看護師 転職」で検索した時にGBPが表示されれば、サイトのSEOより遥かに効果がある。住所非公開でも登録可能。

### 4-4. Mac Mini依存からの脱却

GitHub Actionsへのcron移行を検討すべき。少なくとも以下は外部化可能:
- ハローワーク求人取得（GitHub Actions + cron）
- レポーティング（GitHub Actions or Cloudflare Workers Cron Triggers）
- コンテンツ生成キューの管理（D1/KVに完全移行）

## 5. 強み・活かせるポイント

1. **技術的完成度は極めて高い。** 個人プロジェクトとしてはありえないレベルの自動化基盤。24,488施設DB、2,936求人DB、Haversine距離計算付きマッチング、4段階AIフォールバック、Slack双方向連携は、大手エージェントのIT部門でも数ヶ月かかる仕事
2. **手数料10%は刺さる訴求。** 病院側の採用コスト削減は明確なペインポイント。fee-comparison.htmlの内容は説得力がある
3. **ハローワーク求人の自動取得パイプラインは本物の差別化。** 毎朝06:30に3,600件を取得→ランク付け→D1投入。これを手数料10%の病院営業と組み合わせれば強い
4. **LINE Botのhandoff設計は秀逸。** 「電話は控えた方が良いですか？」の分岐、緊急キーワード検出→即Slack通知は、看護師のペインを理解している証拠
5. **コスト構造が極めて軽い。** CF Workers AI（無料）+ GitHub Pages（無料）+ Mac Mini 1台。月間ランニングコストが¥15,000以下で事業インフラが回るのは、大手に対する構造的優位

## 6. 項目別の詳細所見

### A. ビジネスモデル

**10%紹介手数料の妥当性:**
- 業界標準20-35%に対して破壊的。看護師1人（年収450万）あたり45万円 vs 大手90-157万円
- 月1件成約で45万円。Mac Mini + Claude Code + ドメインで運営コスト月5万円以下なら、粗利40万円/月
- **ただし、この計算は「病院と紹介契約を締結している」前提。** その契約がゼロなら収益もゼロ

**市場規模:**
- 関東4都県（東京・神奈川・千葉・埼玉）の看護師は約35万人。年間転職率10%として3.5万人/年
- 市場規模感は十分。ただし大手（レバウェル、ナース人材バンク、マイナビ看護師）が圧倒的な広告費で支配
- 県西エリアのみでは市場が狭すぎる。関東4都県に拡大したのは正しい判断

**差別化:**
- 手数料10%は明確。ただし「安かろう悪かろう」の懸念を払拭する実績がない
- 「AI転職」は訴求としてはキャッチーだが、看護師がそれを求めているかは不明。看護師は「信頼できる人に相談したい」のであってAIに相談したいわけではない

### B. 集客ファネル

**SNS→LINE→成約の導線:**
- TikTok 9本投稿で3,500再生/週。Instagram 21本投稿。いずれもLINE登録への変換は0
- SNSコンテンツは「あるある系」が35%。看護師は共感するかもしれないが、「この人に転職を任せたい」とは思わない
- **ファネルの入口（認知・興味）と出口（信頼・行動）の間に巨大なギャップがある**

**LP訴求力:**
- メインLP（lp/job-seeker/index.html）のヘッド部は適切。手数料10%、電話なし、無料の訴求が明確
- ただしJSON-LD Organization の住所が「愛知県岩倉市」。神奈川ナース転職なのに愛知。看護師がdevtoolsを見ることはないが、Google はこれを見る
- ファーストビューにLINE登録ボタンがあるのは良い

**ナーチャリング:**
- LINE Bot上のnurture_warm/cold stateは「実装予定」のまま未実装
- 一度離脱した看護師を再度引き戻す仕組みがない
- LINE登録後のプッシュ通知シナリオが存在しない

### C. SEO / コンテンツ

**87ページの内容品質:**
- area/21ページ: テンプレート量産感が強い。「○○市の看護師求人・転職情報を網羅」が全ページで繰り返され、差別化情報が薄い
- guide/41ページ: テーマ選定は良い（fee-comparison, salary-comparison, first-transfer等）。ただしdatePublishedが大半「2026-02-20」で、全て同日に大量生成された印象
- blog/18記事: 構造化データ（FAQ, BlogPosting, BreadcrumbList）は充実。ただしblogとguideの棲み分けが曖昧

**キーワード網羅性:**
- 「神奈川 看護師 転職」系はarea/でカバー
- 「看護師 紹介手数料」「看護師 転職 はじめて」等のロングテールも存在
- **ただし「看護師 転職 口コミ」「看護師 転職 失敗」等のネガティブ系・体験談系が皆無**

**内部リンク:**
- パンくずリスト（BreadcrumbList）は全ページに設置
- 関連記事リンクが一部の記事に存在するが、体系的な内部リンク戦略は見えない
- sitemapは87URL。lastmod一括更新はGoogleに「低品質な大量更新」と判定されるリスク

**タイトル・メタディスクリプション:**
- 全ページに個別設定あり。品質は中〜上。「ナースロビー」のブランド名が全タイトルに入っている
- 一部ページでdescriptionが長すぎる（150文字超）

### D. 技術スタック

**Mac Mini + Claude Code + cron:**
- リスク: 停電、macOS自動更新、ディスク故障、ネットワーク障害で全cron停止
- cron間の依存関係が暗黙的（例: pdca_ai_marketing.sh 06:00で生成→cron_carousel_render.sh 07:30でレンダリング→pdca_sns_post.sh 12:00以降で投稿）
- watchdog.pyが30分間隔で監視しているが、watchdog自体がMac Miniのcronで動いている（番犬も同じ家に住んでいる）

**LINE Bot応答品質:**
- Cloudflare Worker上の実装は堅実。状態管理、フォールバック（OpenAI→Claude→Gemini→Workers AI）、エラーハンドリングが丁寧
- **ただし、これだけ高機能なBotを使っている看護師がゼロという事実**

**セキュリティ:**
- .envにシークレットを管理。git管理外は正しい
- SQLインジェクション対策（バインドパラメータ化）は実施済み
- **ただし、instagram_engage.pyがInstagramアカウントのユーザー名・パスワードを.envから読み取って非公式APIにログインしている。BANされた場合、Instagramアカウント自体が凍結されるリスク**
- `claude --dangerously-skip-permissions` をcronで毎日実行している。権限スキップの名前通り、このプロセスは何でもできる

### E. 求人データ

**ハローワーク求人3,600件:**
- データソース: 厚労省テイキョウAPI（公開API）。信頼性は高い
- 毎朝06:30に自動取得。鮮度は良い
- ランク付け: S:18, A:477, B:951, C:818, D:982。合理的な分布
- **ただし、ハローワーク求人は公開情報。看護師はハローワークで直接応募できる。人材紹介会社を通す価値がない**

**施設DB 24,488件:**
- 厚労省公的データ5ソース。品質は高い
- 病院1,498件の診療科100%、最寄駅99.5%は立派
- **ただし、この施設DBと紹介手数料10%の契約は紐づいていない。DB上の病院が「ナースロビー経由の紹介を受け入れる」とは限らない**

**関東4都県に絞った場合:**
- D1 facilities: 24,488件（東京12,748/神奈川5,165/埼玉3,673/千葉2,902）
- D1 jobs（ハローワーク）: 2,936件
- **病院側の受け入れ体制は確保済み。看護師が来れば即紹介可能**

### F. 広告運用

**Meta広告:**
- v6（トラフィック目的）: ¥6,108消化→0登録。AN/インストリームにゴミ配信+LINE直リンク不通。失敗原因は特定済み
- v7（リード目的）: 3クリエイティブ（動画1+静止画2）、IG/FBフィード手動4面のみ。**まだ出稿していない**
- 設計は改善されている（LP経由、リード目的、手動配置限定）が、実行されていないので評価不能

**クリエイティブ:**
- 3本（年収診断型/共感型/数字インパクト型）は適切な軸
- 全て「電話なし・無料・ブロックOK」明記は良い。看護師の電話恐怖は本物
- **ただし、日予算¥2,000では統計的に有意なA/Bテストは不可能。最低でも¥5,000/日×2週間は必要**

## 7. 推奨アクションTOP5（優先順位付き）

### #1. 今日中にMeta広告v7を出稿する
**なぜ:** 技術はもう十分。あとは人を連れてくるだけ。1日でも早く。
**手順:**
1. Meta Ads Managerで「NR_2026-04_line_direct」キャンペーン作成（リード目的）
2. 日予算¥3,000（¥2,000では学習フェーズが長すぎる）
3. ターゲット: 24-38歳女性、関東4都県、看護師関連インタレスト
4. 配置: Instagram Feed + Facebook Feed のみ（AN/インストリーム除外）
5. LP経由URL: 既存の `/api/line-start?source=meta_ad&intent=direct`
6. 3クリエイティブを1広告セットに入れ、Meta側にCBO最適化させる
7. **3日後に中間確認、7日後にCPA評価**

### #2. 集客チャネルをSNS/SEO/広告の3本に絞り、各チャネルにKPIを設定する
**なぜ:** 病院側の受け入れ体制は既にあり、看護師集客が唯一のボトルネック。集客に直結しない作業を全て止める。
**手順:**
1. 広告: Meta広告v7で週次CPA追跡。目標CPA ¥5,000以下
2. SEO: インデックス率20%→80%を最優先。Search Consoleで手動インデックス登録リクエスト（優先10URL）
3. SNS: Instagram/TikTokの「プロフィールリンククリック数」をKPIに設定。再生数ではなくクリック数
4. 各チャネルの「LINE登録1件あたりコスト」を週次で比較し、最もCPAが安いチャネルに予算集中
5. **集客に直結しないcron（autoresearch, competitor等）は一旦停止**

### #3. cron autoresearchを停止し、コストを広告に回す
**なぜ:** 月¥5,000〜15,000のClaude CLI消費は、看護師がゼロの段階では純粋な浪費。
**手順:**
1. `crontab -e` で `cron_autoresearch.sh` の行をコメントアウト
2. `pdca_weekly_content.sh` もClaude CLI認証エラーで失敗し続けているのでコメントアウト
3. 浮いた予算をMeta広告の日予算上乗せに充てる
4. LINE登録が月30件を超えた段階でautoresearchを再開

### #4. Googleビジネスプロフィールを登録する
**なぜ:** 「後回し」リストに入っているが、ローカルSEOで最もROIが高い施策。
**手順:**
1. GBPに「ナースロビー」で登録（住所非公開OK、許可番号公開OK）
2. カテゴリ: 「人材紹介」
3. サービスエリア: 東京都、神奈川県、千葉県、埼玉県
4. 写真・投稿を週1回更新
5. Googleレビューが1件でもつけば、ローカル検索での表示確率が大幅に上がる

### #5. SEOコンテンツを「体験談」で差別化する
**なぜ:** 87ページのAI生成コンテンツは、他の転職サイトも同じことをやっている。差別化できるのは一次情報だけ。
**手順:**
1. 代表者（または関係者）が看護師3人にインタビュー（匿名可）
2. 「30代看護師が小田原で夜勤なし転職した体験談」等の記事を3本作成
3. 実名顔出しが無理なら、イニシャル+年代+エリアの匿名体験談でもE-E-A-Tは大幅に改善
4. 体験談ページにJSON-LD `Review` を追加
5. 既存の地域ページから体験談ページへ内部リンク
