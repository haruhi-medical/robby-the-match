# ロビーLINE Bot 関東圏シミュレーション検証

## 目的
対応エリア（東京/神奈川/千葉/埼玉）内でのLINE Bot導線品質を徹底検証する。

## 件数
- 実行エージェント: 4名
- 各エージェント: 50ケース
- 合計: **200ケース**
- 対象: 東京都/神奈川県/千葉県/埼玉県

## 役割
| 役割 | 担当 | 責務 |
|------|------|------|
| Manager | 管理 | 計画・割当・集計・報告 |
| Auditor | 監視 | 全件監査・重複検知 |
| Worker 1 | 東京都 | 50ケース |
| Worker 2 | 神奈川県 | 50ケース |
| Worker 3 | 千葉県 | 50ケース |
| Worker 4 | 埼玉県 | 50ケース |

## 検証観点（全国版で発覚した対応エリア内の問題を深掘り）
1. サブエリア選択の網羅性（東京23区/多摩、神奈川6エリア）
2. 施設マッチング精度（D1 DB 24,488件の検索品質）
3. 千葉/埼玉のAREA_CITY_MAP漏れ（秩父/熊谷/銚子/館山等）
4. 東京「どこでもOK」のフィルタなし問題
5. 自由テキスト入力の無視問題
6. 専門分野（ICU/NICU/美容/透析）のマッチング品質
7. handoffフローの完全性
8. 広告→LINE直リンク→meta_ad welcomeの動作
9. 境界ケース（県境通勤、複数エリア希望等）
10. adversarialケース（誤字、矛盾条件、離脱復帰等）
