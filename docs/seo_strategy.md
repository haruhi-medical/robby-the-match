### 1-2. 競合の強み・弱み分析

#### 大手の強み（正面衝突すべきでない領域）
- **圧倒的な求人数**: 数万〜数十万件の求人データベース
- **ドメインパワー**: DR（Domain Rating）80以上のサイトが複数
- **都道府県別・市区町村別のページ自動生成**: 数千ページ規模の地域ページ
- **ブランド認知**: テレビCM・Web広告への大規模投資
- **被リンク数**: 数万〜数十万の被リンク

#### 大手の弱み（攻撃すべき領域）
- **地域の深掘りが浅い**: 「神奈川県」単位の情報で、県西部の詳細な地域情報は薄い
- **手数料が高い**: 年収の20-30%が相場（病院の9割以上が「高い」と感じている）
- **紹介経由の半年以内離職率が23.1%**: 他ルート（12.0%）の約2倍
- **担当者の質にばらつき**: 成果主義で転職を急かすとの口コミ多数
- **施設側の本音が書けない**: 求人掲載料ビジネスのため施設批判ができない
- **地域のリアルな情報がない**: 通勤事情、生活環境、地域コミュニティ情報が欠如

#### github.ioドメインの競合状況
- **看護師転職関連のgithub.ioサイトは現時点でゼロ**
- GitHub Pagesで看護師転職サービスを展開すること自体がユニークなポジショニングになる
- ただしgithub.ioのドメインパワーは低いため、独自ドメイン取得を強く推奨

## 3. 差別化ポジショニング

### 3-1. ROBBY THE MATCHの独自価値提案（USP）

大手との正面衝突を避け、以下の3軸で差別化する。

#### 軸1: 手数料破壊（施設向け訴求）
- **業界平均20-30%に対して10%** = 施設の採用コストが半分〜3分の1
- 年収400万円の看護師で計算:
  - 大手利用: 80-120万円
  - ROBBY THE MATCH: 40万円
  - **施設の節約額: 40-80万円/人**
- 厚生労働省の手数料公開義務化が追い風

#### 軸2: 地域密着（看護師向け訴求）
- **神奈川県西部に特化** = 大手が手薄な地域の深い情報を持つ
- 地域の病院・施設の内部情報、通勤事情、生活環境を提供
- 県西地域の医療機関ネットワーク（小田原市立病院、山近記念総合病院、足柄上病院等）との関係性

#### 軸3: 転職品質（双方向訴求）
- 低手数料 = 施設側が採用に積極的 = 看護師の採用確率が上がる
- 地域特化 = ミスマッチが減る = 離職率が下がる
- 大手の半年以内離職率23.1%に対する差別化

### 3-2. SEOにおける差別化の実装方法

| 大手ができないこと | ROBBY THE MATCHがやること |
|-----------------|-------------------------|
| 県西部の各市区町村の詳細情報 | 9エリア全ての地域特化ページを作成 |
| 手数料の本音（自社が高いとは書けない） | 手数料比較コンテンツを堂々と公開 |
| 施設側へのコスト削減提案 | 施設向けページで採用コスト削減を具体的に訴求 |
| 地域の生活情報 | 通勤・子育て・住環境情報を統合 |
| 厚労省規制の文脈での自社優位性 | 規制動向をコンテンツに織り込み、信頼性を構築 |

## 5. 内部リンク構造

### 5-1. リンク構造の設計思想

「ハブ＆スポーク型」と「サイロ構造」を組み合わせ、Googleのクローラーが効率的に全ページを巡回でき、かつテーマの関連性が明確に伝わる構造を設計する。

```
                    ┌─────────────────┐
                    │   トップページ   │
                    │    (/)           │

    │ 看護師向け │   │ 施設向け  │   │ 共通      │

    └─────┬─────┘   └─────┬─────┘   └───────────┘
```

### 5-3. 内部リンクルール

#### ルール1: 全ページからトップページへのリンク
- ヘッダーロゴからトップへリンク
- パンくずリストの先頭をトップに設定

#### ルール2: 地域ページ間の相互リンク
- 各地域ページの末尾に「近隣エリアの看護師転職情報」セクションを設置
- 例: 小田原ページ → 秦野、南足柄、平塚へリンク
- 例: 藤沢ページ → 茅ヶ崎、平塚へリンク

```
小田原 ←→ 秦野 ←→ 伊勢原 ←→ 厚木 ←→ 海老名
  ↕          ↕                       ↕
南足柄    平塚 ←→ 茅ヶ崎 ←→ 藤沢
```

#### ルール3: 地域ページからガイド記事へのリンク
- 各地域ページ内で関連するガイド記事へ自然にリンク
- 例: 各地域ページの「給与相場」セクション → 給料比較ページへ
- 例: 各地域ページの「訪問看護」言及箇所 → 訪問看護ガイドへ

#### ルール4: ガイド記事から地域ページへのリンク
- ガイド記事内で「神奈川県西部では...」と言及する際に各地域ページへリンク
- 例: 給料比較ページの「エリア別」セクション → 各地域ページへ

#### ルール5: 施設向けページと看護師向けページの相互リンク
- 手数料比較ガイド（看護師向け） ←→ コスト削減ガイド（施設向け）
- 「施設にとっても手数料が安いことは看護師のメリットになります」という文脈で自然にリンク

#### ルール6: CTAへの誘導リンク
- 全ページのコンテンツ内とページ末尾にお問い合わせ/相談フォームへのCTAリンクを設置
- 看護師向けページ: 「無料相談はこちら」
- 施設向けページ: 「手数料10%の詳細を確認する」

#### ルール7: FAQページへの分散リンク
- 各ページで関連するFAQ項目がある場合、そのFAQへのアンカーリンクを設置
- 例: 手数料比較ページ → FAQ「紹介手数料10%とはどういう意味ですか？」

## 7. テクニカルSEO

### 7-1. GitHub Pagesの制約と対策

| 制約 | 対策 |
|------|------|
| サーバーサイド処理不可 | 静的HTMLで全ページ事前生成 |
| .htaccess使用不可 | Jekyll/GitHub Pagesのリダイレクト機能を活用 |
| HTTPSは自動対応 | カスタムドメインでもHTTPS化可能 |
| サーバーログ取得不可 | Google Analytics 4 + Google Search Consoleで補完 |

### 7-2. 独自ドメイン推奨

**強く推奨**: `robby-the-match.com` または `robby-match.jp` 等の独自ドメインを取得し、GitHub Pagesのカスタムドメイン機能で設定する。

理由:
- `xxx.github.io` ではブランド認知・信頼性が低い
- 独自ドメインはドメインパワーの蓄積が可能
- `.jp` ドメインはGoogleの日本ローカル検索で若干有利

### 7-3. メタタグ最適化

各ページに以下を必ず設定:

```html
<!-- 基本メタタグ -->
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>[ページ固有タイトル] | ROBBY THE MATCH</title>
<meta name="description" content="[120-160文字のページ説明。主要KWを自然に含む]">

<!-- OGP（SNS共有用） -->
<meta property="og:title" content="[タイトル]">
<meta property="og:description" content="[説明]">
<meta property="og:type" content="website">
<meta property="og:url" content="https://[ドメイン]/[パス]/">
<meta property="og:image" content="https://[ドメイン]/images/ogp.png">
<meta property="og:site_name" content="ROBBY THE MATCH">
<meta property="og:locale" content="ja_JP">

<!-- canonical -->
<link rel="canonical" href="https://[ドメイン]/[パス]/">

<!-- 構造化データ -->
<script type="application/ld+json">[JSON-LDをここに挿入]</script>
```

### 7-4. Core Web Vitals最適化

GitHub Pagesは静的ホスティングのため、以下を徹底すればCore Web Vitalsは高スコアを達成しやすい。

| 指標 | 目標値 | 対策 |
|------|-------|------|
| LCP（Largest Contentful Paint） | < 2.5秒 | 画像の遅延読み込み、WebP形式使用、CDN活用 |
| INP（Interaction to Next Paint） | < 200ms | JavaScript最小化、静的HTMLの強み |
| CLS（Cumulative Layout Shift） | < 0.1 | 画像にwidth/height属性指定、フォント表示戦略 |

### 7-5. サイトマップ・robots.txt

```xml
<!-- sitemap.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url><loc>https://[ドメイン]/</loc><changefreq>weekly</changefreq><priority>1.0</priority></url>
  <url><loc>https://[ドメイン]/nurse/odawara/</loc><changefreq>monthly</changefreq><priority>0.8</priority></url>
  <!-- 全ページを列挙 -->
</urlset>
```

```
# robots.txt
User-agent: *
Allow: /
Sitemap: https://[ドメイン]/sitemap.xml
```
