## 0. 原則の再確認

### 0-1. AI が全部やる
- 人間キャリアアドバイザーがやる業務（書類・面接・交渉・退職支援・定着フォロー）を **全てAIが代行**
- 人間が出るのは 2 箇所のみ:
  - **(A) 契約・請求**（法務・商流）
  - **(B) 緊急時**（自殺示唆・パワハラ等 14キーワード検出）
- その他、**病院側への送信直前の「社長承認」**（書類・推薦文・条件交渉文面）は Slack で 1クリック承認。実務の判断はAI

### 0-2. ハンドオフしない
- 4ターンの心理的課題ヒアリング後、そのまま **AI が連続してフェーズを進める**
- Slack通知は「進捗記録」目的のみ。**BOTは沈黙しない**
- 「人と話したい」明示要求時のみ handoff モードへ

### 0-3. 情報収集のリズム
- 一度に 1 問しか聞かない（大手のような尋問を避ける）
- 「必要になった時に、必要な情報だけ聞く」のJust-in-time 方式
- 離脱示唆が出たら追わない（既存ロビーとの差別化）

## 2. 質問・応答・遷移の完全定義

以下、各フェーズについて:
- **目的**: 何を確定させるか
- **AI の発話（テンプレ）**: 実際に送るメッセージ
- **想定される返答パターン**
- **分岐条件**
- **保存先**（D1カラム / KV）
- **離脱・拒否時**

### フェーズ 7: 求人マッチング提示（phase: MATCHING）

**目的**: プロファイル情報を元に、D1 求人DBから上位3-5件を提示

**裏ロジック**:
1. 既存 `nurse-robby-db` の jobs テーブルから `area`, `facility_type`, `workstyle` で基本フィルタ
2. Turn 4 の「譲れない条件」を重み付きスコアに加算:
   - 「子育て理解」→ 託児所あり/有給消化率90%+/時短可
   - 「夜勤減らしたい」→ 月夜勤3回以下
   - 「人間関係」→ 定着率高・評判良
3. 上位3件を抽出（Flex Message カルーセル）

**AI発話（ブリッジ）**:
```
ありがとうございます。
{displayName}さんの条件に合う求人を神奈川県内から
お探ししました。

特にマッチ度が高い3件をご紹介します。
```

**Flex Message（3カード）**: 各カードに以下を表示
- 施設名（or 仮名）
- 月給レンジ
- 勤務形態
- 所在地・最寄駅
- AI所見（「月夜勤2回実績・有給消化率92%」等）
- ボタン: [詳しく見る] [気になる]

**フッターメッセージ**:
```
気になる求人はありましたか？
詳細を見る、他を見る、質問する、どれでも大丈夫です。

◇ 1番の求人を詳しく
◇ 2番の求人を詳しく
◇ 3番の求人を詳しく
◇ 他の求人も見たい
◇ 条件を少し変えたい
◇ この中の何かについて質問したい
```

保存: `candidates.phase = JOB_QA`, `candidates.presented_jobs = [job_ids]`

### フェーズ 8: 求人Q&A（phase: JOB_QA）

**目的**: 求人詳細について、施設DBから事実ベースで質問に答える

**ユーザー想定質問**:
- 「この病院、月夜勤何回ですか？」
- 「残業はどれくらい？」
- 「有給取れますか？」
- 「教育体制は？」
- 「面接は厳しいですか？」
- 「求人票に書いてないこと教えて」

**AI応答ルール**:
- 施設DBに事実として記載がある → 事実を返答（出典明示: 「厚労省R5病床機能報告より」等）
- 情報がない → 「その情報は弊社に記録がありません。先方に確認のうえ、1時間以内にお戻しします」+ Slack通知（社長が病院確認 → `!aica reply` でAI発話に反映）
- 推測・「たぶん」は禁止

**応答テンプレート（情報あり）**:
```
ご質問ありがとうございます。
田中クリニックの月夜勤回数は{回数}回です（令和5年病床機能報告より）。

他に気になる点はありますか？
```

**応答テンプレート（情報なし）**:
```
ご質問ありがとうございます。
その点は弊社データに記録がないため、
先方に確認のうえ1時間以内にお戻しします。

その間、他の質問がありましたらお聞きください。
```

**分岐（ユーザー発言の意図判定）**:
- 「応募したい」「進めたい」「この求人で」 → APPLY_CONFIRM
- 「他の求人」「他も見せて」 → MATCHING_MORE（次の3件）
- 「やめておく」「考える」 → PAUSED（沈黙・Day 3 にPush再開）

### フェーズ 12: AI書類生成・提示（phase: DOCUMENTS_GEN）

**目的**: 履歴書・職務経歴書・志望動機書をAI生成しPDFで送付

**裏ロジック**:
1. GPT-4o に `profile_json` + `work_history` + Turn1-4の要約 + 応募先情報を投入
2. 3つの文書を生成:
   - 履歴書（JIS規格準拠）
   - 職務経歴書（1000字以内）
   - 志望動機書（600字以内、応募先特化）
3. `pdf-lib` でPDFレンダリング（Cloudflare Workers内）
4. 一時URLでLINEに送信（Cloudflare R2 に1時間TTLで保存）

**AI発話（生成中）**:
```
作成中です…（10〜30秒）
```

**AI発話（完成時）**:
```
書類3点が完成しました。

◇ 履歴書
◇ 職務経歴書
◇ 志望動機書（{病院名}向け）

[履歴書を見る]
[職務経歴書を見る]
[志望動機書を見る]
[まとめて確認]

内容を確認してください。
修正したい箇所があれば「ここを直して」と教えてください。
問題なければ「これで」とお伝えください。
```

遷移: DOCUMENTS_GEN → DOCUMENTS_REVIEW

## 3. 離脱・特殊ケース共通ルール

### 3-1. 沈黙時の挙動

| 状況 | 対応 |
|------|------|
| 1時間無応答 | 何もしない（ミサキが夜勤等の可能性） |
| 24時間無応答 | 1回だけ控えめなPush「お時間のある時にどうぞ」 |
| 3日無応答 | 1回だけPush「{前回の会話の続き}からでも始められます」 |
| 7日無応答 | 沈黙モード（それ以降cronは停止） |
| 14日無応答 | `candidates.phase = PAUSED` にセット |

### 3-2. 「人と話したい」明示要求

AI発話:
```
承知しました。
弊社の人間担当者にお取り次ぎします。

担当者のレスポンス時間: 平日9-18時は1時間以内、それ以外は翌営業日です。
お急ぎの場合は「急ぎ」とお送りください。
```

→ `candidates.phase = HANDOFF_TO_HUMAN`, BOT沈黙、Slack `#aica-handoff` に通知

### 3-3. 緊急キーワード検出（既実装、維持）

14キーワードのいずれかを検出 → いのちの電話案内 + Slack `#aica-urgent` に通知 → BOT沈黙

## 6. 本書の使い方

- 各フェーズ実装時、本書の該当セクションの発話テンプレ・分岐を実装に落とす
- 社長は定期的に各フェーズを通しで試して、発話の温度感・質問順・分岐を確認
- 発話文言の改善は本書を先に更新、そのあとコード反映
- 離脱率・完遂率を phase別に計測 → ボトルネック特定 → 文言改善の autoresearch ループへ
