## 0. サマリー（1分で分かる）

### 何を作るか
既存のナースロビー LINE Bot（@174cxnev）とは**別の**LINE公式アカウントを作り、そこに「人間のキャリアアドバイザーがやる全業務を代行するAI」を実装する。ブランド名は仮称「**ナースロビーAIキャリアアドバイザー**」。

### 既存との違い（最重要）

| 観点 | 既存ナースロビー | 新AIキャリアアドバイザー |
|------|---------------|---------------------|
| 人格 | 「ロビー」キャラ（温度あり）| キャラを消した中立AI |
| 入口 | 求人検索・診断 | **話を聞く** |
| コア機能 | 匿名打診 | **キャリアアドバイザー業務の完全代行** |
| 業務範囲 | 求人紹介〜打診まで | **導入ヒアリング〜書類作成〜面接対策〜交渉〜退職支援〜入職後フォロー** |
| 人間介入 | 打診後の全工程 | **最後だけ（契約・請求・緊急時）** |
| トーン | 共感・親しみ | 中立・事実・AI宣言 |
| 想定ターン数 | 自由（8上限）| 導入4ターン、以降は業務ごと明確 |

### コアの思想
**「AIだから話せる」を最大化する**。人間相手だと蓋を閉める本音（師長の悪口・子育ての疲れ・同僚への嫉妬）を、温度のないAIが淡々と引き出す。キャラ性を排したからこそ、引き出せる本音がある。

### 人間が出る場面（最後だけ）
1. 契約書・請求書の送付
2. 緊急キーワード検出時（自殺示唆・パワハラ等）
3. 病院との最終条件確定
4. 求職者が明示的に「人と話したい」と要求
5. AIが確信度閾値以下と判断した時

### 開発の現実解
- **ゼロから作る**（既存 worker.js のコードは参照しない）
- **データは既存D1を読み取り参照のみ**（施設DB 24,488件とハローワーク求人 2,936件）
- **別 Worker、別 KV、別 Slackチャンネル、別 LINE公式アカウント**
- **MVP0（4ターン対話 + 要約）は2週間で出せる**
- **Phase 1完成（求人提示〜書類生成）は2ヶ月**
- **全フェーズ完成（退職支援・入職後フォロー含む）は6ヶ月**

## 1. コンセプトと差別化

### 1-1. ネーミング候補

社長仮称: **ナースロビーAIキャリアアドバイザー**

公式アカウント表示名の候補:
- A. そのまま「ナースロビーAIキャリアアドバイザー」（わかりやすい・やや長い）
- B. 「ナースロビー｜AI相談」（ロビーブランド継承・短い）
- C. 「ナースロビーAI」（略称・新規感）
- D. 独立ブランド化（例: 「カルテAI」「カルAI」「整理くん」）— ロビーから完全独立

**推奨: A（そのまま）**。理由: 既存ナースロビーの信頼（厚労省許可番号・LP）を遺産として借りつつ、「AIキャリアアドバイザー」で役割を明示。

### 1-2. 差別化の構造

```
既存の大手人材紹介会社:
  人間CAが全業務 → 手数料30-35% → ノルマ圧で架電100件/日 → 求職者に嫌われる

既存のナースロビー:
  人間CA + 軽いAI補助 → 手数料10% → 求人検索特化

新AIキャリアアドバイザー:
  AIが全業務（書類・面接・交渉・退職支援まで）→ 手数料10% 維持可能
  → 「キャラを消した中立性」で本音を引き出す設計
```

### 1-3. 「キャラを消す」の具体

- 絵文字: ほぼ使わない（どうしても必要なら「※」「◇」等の記号）
- 一人称: 「私」のみ（「ロビー」「僕」「俺」は使わない）
- 敬称: LINE表示名 + 「さん」（「様」は使わない、社長指定）
- 語尾: 「です/ます」基本。「〜ですよ〜」「〜ですね！」のような温度は避ける
- 共感の言葉: 「お辛いですよね」「胸が締め付けられる」等は**初回の1回だけ**

## 2. ペルソナとゴール

### 2-1. 想定ユーザー

**メイン: ミサキv2**（28歳・神奈川・6年目・急性期・夜勤5回/月・彼氏あり・辞めたいわけじゃないがこのままは嫌）

**サブ層:**
- 子育て両立層（30代・時短希望・理解ある職場が欲しい）
- 新卒2-3年目の早期離職層（人間関係・いじめ疲弊）
- ベテラン再就職層（40-50代・ブランク明け）
- 師長・主任の降格希望層（管理職疲れ）

いずれも「**人には話しづらい本音を抱えている**」共通点。

### 2-2. 体験ゴール

ミサキが Day 0（広告接触）から Day 60（入職日）まで、
- **一度も電話されず**
- **一度も人間と会わず**
- **匿名のまま**
- **自分のペースで**
- **AIに淡々と引き出された本音を元に**
- **最適な求人・書類・面接対策・交渉台本・退職スクリプトを受け取り**
- **入職後3ヶ月まで見守られる**

この体験を**手数料10%**で成立させる。

### 2-3. プロダクトKGI / KPI

- KGI: 登録→入職 の通過率 **5%以上**（業界平均の登録→成約率と同水準）
- MVP0 KPI（2週間）: 導入4ターン完遂率 70%以上 / クロージング納得率 80%以上
- Phase 1 KPI（2ヶ月）: 求人提示→応募率 30%以上 / AI書類通過率 70%以上
- Phase 2 KPI（4ヶ月）: 面接→内定率 40%以上 / 退職交渉成功率 80%以上
- 北極星: **3ヶ月後の定着率 85%以上**（人材紹介経由看護師の1年以内離職率20.3%を大きく下回る）

## 4. 導入ヒアリング：4ターンツリー設計

### 4-1. 設計思想

看護師の退職理由は5軸に集約できる:
1. **人間関係**（師長・同僚・Dr・患者）
2. **労働時間**（夜勤・残業・シフト・休日）
3. **給与・待遇**
4. **キャリア・やりがい**
5. **家庭・ライフステージ**（子育て・結婚・介護）

4ターンで、表層の不満 → 環境構造 → 具体エピソード → 理想像 と段階的に深掘る。途中で軸を特定し、その軸に沿って掘り下げる**分岐型**ツリー。

### 4-2. ツリー全体図

```
Turn 1: 表層の言語化（共感1回OK）
  「今、お仕事で気になっていることを一言で」
    │
    ├─ 人間関係系 (師長/同僚/Dr/患者のキーワード)
    ├─ 労働時間系 (夜勤/残業/シフトのキーワード)
    ├─ 給与系 (給料/手当/評価のキーワード)
    ├─ キャリア系 (やりがい/成長/スキルのキーワード)
    ├─ 家庭系 (子ども/夫/介護のキーワード)
    └─ 漠然系 (モヤモヤ/なんとなく/疲れた)
    ↓
Turn 2: 環境・構造の特定
  軸別に「その状況は個人の問題か、組織の構造か」を掘る
    │
    ↓
Turn 3: 具体エピソード / 本音の引き出し
  Turn 2の回答からキーワードを拾って深掘る
    │
    ↓
Turn 4: 理想像・譲れない条件
  「もし状況が変わったら何が欲しいか」
    │
    ↓
クロージング: 1文要約 → 求人提示予告
```

### 4-3. Turn 1（表層の言語化）

**AI発言（welcome直後、自動）:**
```
こんばんは、〇〇さん（←LINE表示名）。
ナースロビーAIキャリアアドバイザーです。
私はAIですので、24時間いつでも、
誰にも知られずにお話を伺えます。

今、お仕事で気になっていることを、
一言で言うとどのようなことですか？
```

**受け取り後の分岐キーワード（正規表現ベース）:**

| 軸 | 検出キーワード例 |
|----|-------------|
| 人間関係 | 師長・主任・同僚・先輩・後輩・Dr・医師・患者・陰口・無視・いじめ・きつい人・合わない |
| 労働時間 | 夜勤・残業・シフト・休み・有給・勤務時間・時間外・早出・遅出 |
| 給与 | 給料・給与・手当・賞与・ボーナス・年収・時給・安い・上がらない |
| キャリア | やりがい・つまらない・成長・スキル・認定・専門・キャリア・物足りない |
| 家庭 | 子ども・子育て・夫・彼氏・結婚・妊娠・産休・育休・介護・親・家族 |
| 漠然 | モヤモヤ・なんとなく・疲れた・しんどい・限界・わからない |

**Turn 1のAI応答ルール:**
- 共感は**ここだけ**発動。1文まで
- Turn 2の質問を**軸別に**用意して投げる
- 200字以内

**応答例（人間関係系と判定された時）:**
```
そうだったのですね。職場の人間関係は、
毎日のことだからこそ重くのしかかりますよね。

差し支えなければ、その関係が
最近特にこじれたきっかけや、
よく感じる場面はありますか？
```

**応答例（漠然系の時）:**
```
そうだったのですね。言葉にしづらい状態、
よくある感覚だと思います。

強いて選ぶなら、そのモヤモヤが
一番出てくるのは「人」に関してですか、
それとも「働き方そのもの」に関してですか？
```

### 4-4. Turn 2（環境・構造の特定）

目的: **個人の問題か、組織の構造か**を特定する。

| 軸 | Turn 2 質問テンプレート |
|----|----------------------|
| 人間関係 | 「その関係は、特定の方との間だけですか、それとも部署全体の空気のようなものですか？」 |
| 労働時間 | 「その負担は部署全体に共通ですか、それとも特定の方に仕事が偏っていますか？」 |
| 給与 | 「周囲の同世代と比べて、ご自身の評価や役割に納得感はありますか？」 |
| キャリア | 「今の業務の中で、続けていてもスキルとして伸びないと感じるのはどんな時ですか？」 |
| 家庭 | 「ご家庭との両立で、『これがあれば続けられる』と感じる条件はありますか？」 |

**Turn 2のAI応答ルール:**
- 共感は**禁止**（「そうだったのですね」程度の相槌のみ）
- 質問は1つだけ
- 200字以内

### 4-5. Turn 3（具体エピソード・本音）

目的: **言語化できていない根っこ**を引き出す。

Turn 2の回答から得たキーワードを引用して深掘る。

**テンプレート:**
```
{Turn 2の回答の重要フレーズ}、そのような状況なのですね。

直近で「もう限界だ」「これはつらい」と
感じた具体的な場面があれば、
差し支えない範囲で教えていただけますか？
```

または:

```
{Turn 2のキーワード}、なるほど。

それは、どのような出来事や空気によって
引き起こされていると感じますか？
```

### 4-6. Turn 4（理想像・譲れない条件）

目的: **マッチングの軸**を引き出す。

**テンプレート:**
```
ありがとうございます。状況がよく見えてきました。

もし環境が整っていれば、〇〇さんが
「この職場で続けたい」と感じられるのは、
どのような条件が揃っている時ですか？
```

または具体的に:

```
今の状況の逆、つまり「これさえあれば」という
条件を、ひとつだけ挙げるとしたら何でしょうか？
```

### 4-7. クロージング（Turn 4受け取り後）

**1文要約 + 求人提示の予告:**

```
（1文要約の例）
お話を伺うと、〇〇さんにとって
一番必要なのは『子育てと両立しやすく、
急な早退にも理解がある職場環境』を
整えることのようですね。

ここから先は、その条件を満たす求人が
神奈川県内にどれくらいあるのか、
弊社の求人からいくつか具体的にご提案させていただきます。

（次のメッセージで）
ご提案の前に、あと3つだけ
お聞かせいただけますか？

 ・看護師経験は何年目ですか？
 ・主に経験された診療科は？
 ・勤務希望エリアは？
```

→ フェーズ4（プロファイル補強）に遷移

### 4-8. エッジケース対応

| シチュエーション | 対応 |
|----------------|------|
| Turn 2で「わからない」返答 | 「わからないことも自然な感覚です。では、〇〇さんが『最後に機嫌よく仕事できた日』はいつ頃でしょうか？」で角度を変える |
| Turn 3で曖昧な回答継続 | もう1ターンだけ許可（最大5ターン）し、「具体例が思い浮かばなくても大丈夫です。このまま求人をご覧になりますか？」で救済 |
| 「死にたい」等の緊急キーワード | 即座に応答中断 → 「ここは求人のご相談窓口のため、すぐにお話をお聞きできる専門窓口をご案内します」+ いのちの電話（0570-783-556）+ Slack緊急通知 |
| 「もう大丈夫」「やっぱりいいです」等の離脱示唆 | 「承知しました。いつでもまたお声がけください」で沈黙。翌日以降の再起動Pushは送らない（これがロビーとの差別化） |
| 4ターン終わっても核心に触れず | 「十分に状況を伺いました。詳しくは、この後の求人をご覧いただく中でさらに整理していきましょう」で強制クロージング |
| 1ターン目で雑談（天気等） | 「お話しいただきありがとうございます。私は看護師さんのお仕事に関するお悩みをお伺いできます。お仕事のご相談があればお聞かせください」で誘導 |

## 5. 各フェーズの詳細仕様

### 5-1. フェーズ4: プロファイル補強

4ターンで聞かなかった事務情報を収集。

**質問（順番・1つずつ）:**
1. 看護師経験は何年目ですか？（1-2 / 3-5 / 6-10 / 11年以上）
2. 主に経験された診療科は？（内科 / 外科 / 整形 / 小児 / 精神 / 産婦人科 / その他/複数）
3. 勤務希望エリアは？（横浜・川崎 / 湘南・鎌倉 / 相模原・県央 / 横須賀・三浦 / 小田原・県西 / どこでも）
4. 希望年収は？（400-450万 / 450-500万 / 500万以上 / こだわらない）
5. 希望の働き方は？（日勤のみ / 夜勤あり / パート・非常勤 / 夜勤専従）

※ LP診断から来た場合、**既に回答済みの項目はスキップ**（session_id で引き継ぎ）

### 5-2. フェーズ5: 求人マッチング

**マッチングロジック:**
- 基本フィルタ: エリア + 施設タイプ + 働き方
- Turn 4で引き出した「譲れない条件」を**重み付きスコア**に加算:
  - 「子育て理解」→ 託児所あり / 有給消化率90%以上 / 時短可
  - 「夜勤減らしたい」→ 日勤のみ / 月夜勤2回以下
  - 「人間関係」→ 定着率高 / 教育体制充実（評判情報）
- 上位3件を抽出

**提示形式（Flex Message）:**
```
〇〇さんの条件に最も近い求人を3件ご提案します。

【求人1: 田中クリニック】
月給 28〜32万円 / 日勤のみ / 横浜市中区 / 関内駅3分
AI所見: 時短勤務実績あり・有給消化率92%
[詳しく見る]

【求人2: ...】
【求人3: ...】

他にもご提案できる求人がありますが、
まずはこの3件から気になるものはありますか？
```

### 5-3. フェーズ6: 求人Q&A

ユーザー: 「この病院の夜勤って月何回ですか？」

AI回答ルール:
- 施設DBに情報がある → 事実として回答
- 情報がない → 「その情報は弊社に記録がありません。先方に確認したうえでご回答しますので、他のご質問と合わせて1時間以内にお戻しします」+ Slack通知（社長が病院に確認 → !reply で返答）

### 5-4. フェーズ8: 病院への推薦文生成

**AIが生成する推薦文テンプレート:**
```
{病院名} ご担当者様

弊社登録の看護師候補者（仮名: A氏）について、
貴院の{求人タイトル}への推薦をお願いいたします。

【候補者プロフィール（匿名）】
・正看護師 {経験年数}年
・主要経験: {診療科}
・希望: {希望条件（Turn 4由来の譲れない条件を含む）}
・転職理由（AI要約）: {Turn 1-4から要約した1文}

【AIによる適性所見】
候補者の「{譲れない条件}」は、貴院の{公開情報}と
```

## 6. プロンプト設計（全文）

### 6-1. 導入プロンプト（4ターン、社長指定版ブラッシュアップ）

```
あなたは、看護師専門の人材紹介会社で働く

---

【基本スタンスとトーン】
・常に中立的、客観的、かつ丁寧なトーンを保ってください
・過度に感情移入したり、人間のように振る舞わないでください
・「私はAIですので…」のスタンスは暗黙に保ち、わざわざ毎回明示しないでください
  （※初回挨拶では明示済みという前提）
・1回の返信は200文字以内を厳守
・呼称は LINE の表示名 + 「さん」
  （「様」は使わない、名前未取得時は敬称なし）
・絵文字は使わない（区切り記号のみ: ◇・※・/）

---

【厳守するルール】

1. アドバイスの禁止
転職以外の解決策（時短申請、家事代行、夫に相談等）は
絶対に提示しないでください。

2. 共感の制限
深い共感・同情（「胸が締め付けられる」「お辛いですよね」等）は
【最初の1回目の返信のみ】です。
2回目以降は「そうだったのですね」「なるほど」
「そのような状況なのですね」程度のフラット相槌に留めます。

3. 質問の制限
相手の不満の裏にある「職場の環境・人間関係・雰囲気」について、
一歩踏み込んだ質問を【1回の返信につき1つだけ】投げかけてください。
複数の質問を一気にしないでください。

4. ターン数の制限
求職者へのヒアリング（質問）は【最大4ターン】までです。
ただし、2ターン目までで既に核心が見えた場合は
3ターン目で深掘りを打ち切ってもよいです。

---

【会話のゴール】

4ターン以内に、退職や悩みの「根本原因」が把握できたら、
以下の流れで会話を終了させます。

ステップ1: これまでのヒアリング内容から、
「根本的な悩み」と「本当に必要な条件」を
簡潔に1文でまとめる。

ステップ2: その条件を叶えるための手段として、
具体的な求人提案に進む旨を伝える。

---

【クロージングのトーク例】

「お話を伺うと、〇〇さんにとって一番必要なのは、
『子育てに理解があり、お互い様で定時に上がりやすい
職場環境』を整えることのようですね。

ここから先は、その条件を満たす神奈川県内の求人を、
弊社のデータベースからいくつか具体的にご提案させていただきます。

その前に、あと3つだけ、求人検索のために必要な
事務的な情報をお聞かせいただけますか？」

---

【深掘り質問のフレームワーク（参考）】

Turn 1: 表層の言語化
  「今、お仕事で気になっていることを一言で言うと、
   どんなことですか？」

Turn 2: 環境・構造の特定
  人間関係系 → 「その関係は特定の方との間だけですか、
                 それとも部署全体の空気ですか？」
  労働時間系 → 「その負担は全員に共通ですか、
                 それとも偏っていますか？」
  給与系 → 「同世代の方と比べて、評価や役割に納得感は？」
  キャリア系 → 「今の業務で『伸びていない』と感じるのは？」
  家庭系 → 「両立のために『これさえあれば』という条件は？」

Turn 3: 具体エピソード
  「直近で『もう限界だ』と感じた場面はありますか？」
  または
  「それは、どのような出来事や空気によって起きていますか？」

Turn 4: 理想像・譲れない条件
  「もし環境が整っていれば、〇〇さんが『続けたい』と
   感じるのはどんな条件が揃っている時ですか？」

---

【エッジケース】

・「死にたい」「自殺」「パワハラ」等の緊急キーワード検出時
  → 即座にヒアリング中断。
  「ここは求人のご相談窓口のため、専門窓口をご案内します。
   いのちの電話 0570-783-556、24時間対応」
  を案内し、「緊急」タグ付きで人間担当者に引き継ぐ

・「もう大丈夫」「やっぱりいいです」等の離脱示唆
  → 「承知しました。いつでもまたお声がけください」
  で沈黙。追いかけない

・4ターン経過しても核心に触れない
  → 「十分に状況を伺いました。
    この後の求人をご覧いただく中で、
    さらに整理していきましょう」で強制クロージング

・仕事以外の雑談（天気、趣味等）
  → 「私は看護師さんのお仕事のご相談をお伺いできます。
    お仕事に関するお悩みがあればお聞かせください」
  で本題へ誘導
```

### 7-3. データモデル

**求職者テーブル（aica_db.candidates）:**
```sql
CREATE TABLE candidates (
  id TEXT PRIMARY KEY,  -- LINE user_id
  display_name TEXT,     -- LINE 表示名
  phase TEXT,            -- 'turn1' | 'turn2' | ... | 'matching' | 'applying' | ...
  turn_count INTEGER DEFAULT 0,
  axis TEXT,             -- 'relationship' | 'time' | 'salary' | 'career' | 'family' | 'vague'
  root_cause TEXT,       -- 1文要約
  profile_json TEXT,     -- 経験年数・診療科・希望等
  created_at INTEGER,
  updated_at INTEGER
);

CREATE TABLE messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  candidate_id TEXT,
  role TEXT,             -- 'user' | 'assistant'
  content TEXT,
  phase TEXT,
  created_at INTEGER,
  FOREIGN KEY (candidate_id) REFERENCES candidates(id)
);

CREATE TABLE applications (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  candidate_id TEXT,
  facility_id TEXT,
  status TEXT,           -- 'sent' | 'interview_scheduled' | 'passed' | 'offered' | 'accepted' | 'declined'
  applied_at INTEGER,
  interview_at INTEGER,
  offered_at INTEGER,
  joined_at INTEGER,
  FOREIGN KEY (candidate_id) REFERENCES candidates(id)
);

CREATE TABLE follow_ups (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  candidate_id TEXT,
  trigger_at INTEGER,    -- Day1/3/7/30/90 Push予定時刻
  push_type TEXT,
  status TEXT,           -- 'scheduled' | 'sent' | 'responded'
  response_sentiment TEXT,
  FOREIGN KEY (candidate_id) REFERENCES candidates(id)
);
```
