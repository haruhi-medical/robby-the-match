## Core Thesis

```
AIは登録率を上げる道具ではなく、スケール可能性を作る装置である。
1人で回せる上限を10件/月から50件/月に引き上げることが本質的価値。
```

**Anti-pattern:** AIでバズる → 登録爆増 → 対応できず破綻
**Correct-pattern:** AIでオペレーション効率化 → 少人数で高品質対応 → 信頼蓄積 → 口コミ拡大

## 現在判明している技術的問題と解決策

### 問題1: Cloudflare Workers AI → Claude API 画像読み込みエラー

**症状:** Cloudflare Workers AIで画像生成は成功（200 OK）するが、
その画像をClaude APIに送ると400 "Could not process image"エラー。

**原因（確率順）:**

1. 画像フォーマットのヘッダー破損（最も可能性高い）
2. Base64エンコード時の破損
3. 極端なアスペクト比や解像度

**解決策:**

```python
# Pillowで一度開き直して正しいPNGとして再保存する
from PIL import Image
img = Image.open("cloudflare_test.png")
img.save("cloudflare_test_fixed.png", "PNG")
# これをClaude APIに送る
```

### 問題2: Cloudflare APIトークン認証エラー（401）

**解決策:**

1. Cloudflareダッシュボード → プロフィール → API Tokens
2. 新規トークン作成（権限: Account - Workers AI: Edit）
3. `~/.zshrc` の `CLOUDFLARE_API_TOKEN` を更新
4. `source ~/.zshrc` で再読み込み

**注意:** `.zshrc` にトークンを書く場合、gitリポジトリに含めないこと。

### 問題3: gpt-image-1の日本語テキスト品質

**推奨ワークフロー（2段階方式）:**

```
Step 1: gpt-image-1 で背景画像のみ生成（テキストなし）
Step 2: Pillow でテキストオーバーレイを後から焼き込む
```

**テキスト後乗せスクリプト例:**

```python
from PIL import Image, ImageDraw, ImageFont

img = Image.open("base_image.png")  # 1024x1536
draw = ImageDraw.Draw(img)

# 日本語フォント（Mac: ヒラギノ、Linux: NotoSansJP）
font = ImageFont.truetype("/System/Library/Fonts/ヒラギノ角ゴシック W6.ttc", 72)

# 半透明黒帯
overlay = Image.new('RGBA', img.size, (0, 0, 0, 0))
overlay_draw = ImageDraw.Draw(overlay)
overlay_draw.rectangle([(0, 600), (1024, 800)], fill=(0, 0, 0, 160))
img = Image.alpha_composite(img.convert('RGBA'), overlay)

# テキスト描画（白文字、中央配置）
draw = ImageDraw.Draw(img)
text = "師長にAIで見せたら黙った"
bbox = draw.textbbox((0, 0), text, font=font)
x = (1024 - (bbox[2] - bbox[0])) // 2
draw.text((x, 650), text, fill="white", font=font)

img.save("slide_1.png")
```

## お前の行動原則

1. **コストを常に意識しろ** — 無料でできることに金を使うな。費用対効果を必ず計算してから実行しろ。
2. **ペルソナで判断しろ** — 「ミサキが見て動くか？」を全ての制作物の最終チェックにしろ。
3. **調べてから動け** — 競合のLP、SNSアカウント、最新の看護師転職市場をWeb検索で確認してから設計しろ。
4. **小さく作って速く出せ** — 完璧なシステムより、今日看護師が見るSNS投稿の方が100倍価値がある。
5. **法律を最優先しろ** — 職業安定法、医療広告ガイドライン、個人情報保護法に違反する可能性が1%でもあるならやるな。
6. **数字で判断しろ** — 「いい感じ」ではなく、KPIで進捗を測れ。
7. **YOSHIYUKIに聞け** — 判断に迷ったら勝手に決めるな。選択肢と推奨案を提示して確認を取れ。

## 禁止事項

- 900行の手順書を書くな（それはv4.1の失敗）
- Reactでフロントエンドを作るな（LPにSPAは不要）
- マッチングアルゴリズムを作るな（月1件の成約に機械学習は不要）
- OpenClawとの双方向オーケストレーションを設計するな（複雑すぎる。まず動くものを作れ）
- 認証情報をコードにハードコードするな
- Lステップを今導入するな（LINE登録月50名超えてから）
- 月3万円以上の新規ツール契約を勝手に決めるな（YOSHIYUKIに確認）
- ベース画像を毎回新規生成するな（使い回してコスト削減）
- ペルソナを無視したコンテンツを作るな（「誰に向けて？」が答えられない投稿は出すな）
