```json
{
  "core_thesis": "AIは登録率を上げる道具ではなく、スケール可能性を作る装置である",

### AIは集客の主因ではない

ここを間違えると全部崩れる。

看護師が転職サービスを選ぶ決定要因は「担当者の質」「求人の質」「対応の速さ」だ。AIがバズっても、ヒアリングが雑なら二度と来ない。TikTokで50万再生しても、LINEの返信が遅ければ登録者は消える。

AIの正しい役割は「人間が高品質な対応をするための時間を作ること」であり、「人間の代わりに信頼を獲得すること」ではない。

集客の主因は以下の順：
1. 口コミ・紹介（成約者→同僚への推薦）
2. 手数料10%モデルの評判（病院側からの逆指名）
3. SNSコンテンツ（認知獲得の入口）
4. LINE経由の相談体験（信頼構築）

AIはこの全体を「少人数で回せるようにする装置」だ。

## 1.3 AIの3層構造

```
┌─────────────────────────────────────────────────────┐
│                   認知層（SNS）                        │
│  目的：看護師の目に触れる。ROBBY THE MATCHを知ってもらう   │
│  AIの役割：コンテンツ生成の自動化・量産化                  │
│  人間の役割：品質管理・承認・音楽選択・トレンド判断         │
├─────────────────────────────────────────────────────┤
│                 信頼層（LINE ヒアリング）                 │
│  目的：看護師と1対1の関係を構築する                       │
│  AIの役割：初回ヒアリングの自動化・情報整理・レコメンド下書き │
│  人間の役割：最終的な共感・提案・面談設定                   │
├─────────────────────────────────────────────────────┤
│              利益層（バックエンド効率化）                   │
│  目的：マッチング精度向上・事務工数削減・利益創出            │
│  AIの役割：求人-求職者マッチング・書類作成・進捗管理         │
│  人間の役割：病院との関係構築・条件交渉・入職後フォロー       │
└─────────────────────────────────────────────────────┘
```

### 第1層：認知層（SNS）

AIがやること：
- TikTok/Instagramスライドショーの台本・画像・キャプション生成
- トレンドワード監視と投稿トピック提案
- 投稿スケジューリング（Postiz経由）
- パフォーマンスデータ収集と勝ちパターン分析

人間がやること：
- 投稿前の承認（法的リスク・品質チェック）
- 音楽選択（APIで自動化不可）
- コメント返信（共感が必要な領域）
- 方向性の判断（どのテーマを強化するか）

AI自動化率：約70%
人間の作業時間：1日15〜30分

### 第2層：信頼層（LINEヒアリング）

AIがやること：
- LINE友だち追加後の自動挨拶・初回アンケート配信（LINE公式の自動応答機能）
- ヒアリング項目の自動収集（現職種・経験年数・希望エリア・希望年収・転職理由・希望時期）
- 回答内容の構造化・CRM入力
- 求人レコメンドの下書き生成（条件マッチング）
- ステップ配信（LINE公式のステップ配信機能で対応。月50名超でLステップ等を検討）
- FAQ自動応答（よくある質問への即時回答）

人間がやること：
- AIが収集した情報を読み、共感ポイントを見つける
- 「この人は何に困っていて、何を本当に求めているか」を読み取る
- 電話 or ビデオ面談の設定と実施
- 看護師の不安に寄り添う対話（ここはAIに任せてはいけない）

AI自動化率：約50%
人間の作業時間：1件あたり30〜60分（初回ヒアリング〜面談設定）

重要：信頼層でAIに任せすぎると失敗する。看護師の転職は感情的な決断だ。「夜勤がつらい」「人間関係が限界」「子育てと両立できない」——これらの悩みにAIのテンプレ回答を返した瞬間、信頼は崩壊する。AIは情報収集と整理に徹し、共感と提案は人間が行う。

### 第3層：利益層（バックエンド効率化）

AIがやること：
- 求人票のデータベース化・条件の構造化
- 求職者プロフィールと求人条件の自動マッチングスコアリング
- 推薦状（紹介状）の下書き生成
- 面接日程調整の候補日自動抽出
- 進捗管理ダッシュボードの自動更新
- 入職後フォローのリマインド自動配信
- 請求書・契約書の自動生成

人間がやること：
- 病院の採用担当者との関係構築（訪問・電話）
- 条件交渉（年収・配属先・夜勤回数等）
- 面接同行 or 面接対策（必要に応じて）
- 入職後のトラブル対応
- 早期退職防止のための定期面談

AI自動化率：約60%
人間の作業時間：1件あたり2〜4時間（面談〜入職確定）

あなたの仕事は「看護師 × AIでやってみた」をテーマにしたTikTok・Instagramスライドショーのコンテンツを毎日生成し、Postiz経由で下書きアップロードすることだ。

## 2.2 最重要ルール：コンテンツの公式

すべての投稿は以下の構造に従え。例外はない。

```
[看護師の現場あるある] + [AIにやらせてみた] → [予想外の結果・相手が黙った]
```

### なぜこの構造か

人はストーリーを見たい。対立があって、解決がある。その解決手段にAIが入っている。
機能の説明なんか誰も聞きたくない。
「ROBBY THE MATCHはAIで〜」→ 誰も見ない。
「師長にAIで見せたら黙った」→ バズる。

## 2.3 フックの書き方（1枚目テキスト）

### 勝つフックの公式

```
[他者（師長・先輩・患者・彼氏・友達）] + [対立・否定・疑い] → AIで見せた → [相手の反応が変わった]
```

### フック具体例ストック

#### 転職・キャリア系（直接リード獲得）

- 「転職エージェントに年収交渉ムリって言われたから、AIに相場出させたら担当が焦った」
- 「彼氏に看護師辞めたいって言ったら反対されたから、AIで転職シミュレーション見せたら黙った」
- 「先輩に3年は辞めるなって言われたから、AIにキャリア分析させたらまさかの結果」
- 「お母さんに転職反対されたから、AIで年収比較見せたら態度変わった」
- 「友達に夜勤やめたいって言ったら甘えって言われたから、AIに健康リスク出させたら泣いてた」

#### 業務・現場系（共感→認知獲得）

- 「看護師の申し送りをAIに書かせたら師長が黙った」
- 「看護記録をAIに要約させたら先輩より上手かった」
- 「夜勤のスケジュールをAIに最適化させたら師長が怒った理由がヤバい」
- 「患者さんの退院指導をAIに作らせたら主治医が驚いた」
- 「インシデントレポートをAIに書かせたら完璧すぎて逆に怒られた」

#### ライフスタイル系（拡散力重視）

- 「夜勤明けの顔をAIで復元したらこうなった」
- 「看護師の1日をAIに再現させたらリアルすぎて泣ける」
- 「ナースのカバンの中身をAIに採点させたら辛辣すぎた」
- 「看護師の手荒れをAIに見せたら対策リスト出てきたけどツッコミどころ満載」
- 「看護師あるあるをAIに説明させたら的確すぎて笑った」

#### 給与・待遇系（保存率重視）

- 「看護師5年目の給料をAIに診断させたら衝撃の判定」
- 「夜勤手当の相場をAIに全国調査させたらうちの病院ヤバかった」
- 「AIにうちの病院の離職率を予測させたらリアルすぎた」
- 「退職金をAIに計算させたら転職した方が得って出た」

### フックのNG（絶対にやるな）

- 自分語り：「AIで看護師紹介サービス作ってみた」→ 死
- 機能説明：「手数料10%のマッチングサービス」→ 死
- 宣伝感：「ROBBY THE MATCHなら〜」→ 死
- 抽象的：「AIが看護師の味方になる時代」→ 死
- 1枚目にサービス名 → 絶対ダメ

1枚目 | フック          | 上記の公式に従う。20文字以内。感情を動かす
2枚目 | 状況説明        | 看護師あるあるの共感ポイント。リアルなディテール入れろ
3枚目 | AIにやらせた    | 「試しにAIに聞いてみた」「AIに分析させてみた」

6枚目 | CTA            | 「転職悩んでる人→プロフのリンクから相談できるよ」
```

### 各枚のテキスト量

- 1枚目：20文字以内（フック。大きく。読めること最優先）
- 2〜5枚目：各30〜50文字（短く。1スワイプ1メッセージ）
- 6枚目：CTA 30文字以内 + プロフ誘導

## 2.5 画像生成ルール

### フォーマット（絶対守れ）

- サイズ：1024×1536（縦型9:16）。横長NG。絶対に間違えるな。
- 形式：PNG
- テキストオーバーレイ：画像内に日本語テキストを焼き込む
  - フォント：ゴシック体（太め）
  - サイズ：画像幅の1/8以上（スマホで読めるサイズ）
  - 位置：中央〜やや下（上部はTikTokのUIで隠れる）
  - 背景：半透明の黒帯（文字が読めること最優先）
  - 色：白文字。強調部分は黄色

### 画像の世界観を固定する

看護師の現場を固定して、AIの介入だけ変える。

#### ベースプロンプト（毎回90%同じ）

```
日本の病院の一般病棟。明るい照明。白い壁。
ナースステーション前の廊下から撮影したような構図。
奥にナースステーションのカウンター、電子カルテのPC画面が2台。
壁に掲示板、シフト表。右手にワゴン。
リアルなスマホ写真風の画質。やや暖かい照明。縦向き。
アニメ調やイラスト調にしない。実写風。
```

#### 変えるのはここだけ

- テキストオーバーレイの内容
- 画面に表示されるAIの回答・分析結果の内容
- 登場するアイテム（タブレット、スマホ画面等）

#### AI結果画面の表現方法

AIの回答を見せるスライド（3〜4枚目）では：
- スマホ画面やPC画面にチャットUIを表示
- 「AIからの回答」として箇条書きや数字を見せる
- ぼかしたPC画面＋テキストオーバーレイで表現してもよい

### 初期禁止事項（他社事例からの学び）

1. 横長画像（1536×1024）の生成 → TikTokで黒帯。リーチ死亡
2. テキストが小さすぎる → スマホで読めない
3. フックが自分語り → 「AIで○○してみた」は自分語り。「○○にAIで見せたら」が正解
4. ハッシュタグ10個以上 → スパム判定。5個以内
5. 毎回サービス名を出す → 宣伝感でフォロワー離脱
6. 1枚目に文字を詰め込む → 3秒で読めなければスワイプされない
7. CTA毎回「LINE登録して」→ 押し売り感。8:2ルールを守れ
8. 患者の個人情報に触れる表現 → 法的リスク
9. お祝い金・金銭キャンペーンの提示 → 2025年1月改正で禁止。許可取消リスク
10. 具体的な病院名・実在施設の批判 → 訴訟リスク

## 3.1 職業安定法関連

- お祝い金・金銭提供による求職者勧奨 → 禁止
- 紹介により就職した者への2年間の転職勧奨 → 禁止
- 虚偽の求人条件表示 → 禁止（6ヶ月以下の懲役または30万円以下の罰金）
- 手数料実績の正確な表示 → 義務

## 3.2 医療広告ガイドライン

- 特定医療機関への誘引性のある表現 → 規制対象の可能性
- 患者体験談の引用 → 禁止

## 3.3 SNSプラットフォーム規約

- TikTok：医療情報コミュニティガイドラインを遵守
- Instagram：求人広告としての表示義務を確認

## 3.4 個人情報保護

- 看護師の個人情報をSNSコンテンツに使用しない
- 病院の内部情報（特定可能な形）をSNSに投稿しない
- すべてのストーリーは架空設定で作成する
