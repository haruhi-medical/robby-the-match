## 2. 体制（計54エージェント）

### 4パネル × 6人 + 議長1人 = 28人

| パネル | ゴール | 6人の専門家 |
|--------|-------|------------|
| **P1 流入** | LP・LINEへの流入量×質の阻害要因抽出 | SEO / Meta広告CRO / TikTokグロース / Instagram成長 / GA4アトリビューション / 競合調査 |
| **P2 コンバージョン** | LP→LINE→診断→ハンドオフの離脱阻害要因 | LP CRO / LINE Bot UX / コピーライター / 診断UX / 離脱分析 / ミサキ本人 |
| **P3 マッチング品質** | 紹介対象の品質とハンドオフ後の成約確率 | D1 DB / マッチングロジック / AI応答品質 / 求人データエンジニア / ハンドオフ品質 / 看護師紹介業務 |
| **P4 基盤** | 事業継続性・コスト・運用健全性 | Worker / cron運用 / Pythonスクリプト監査 / セキュリティ / コスト監査 / SRE |

各パネルに議長1人（統合役）。合計 7 × 4 = 28人。

### 監督2人（二重ゲート）

| 監督 | 役割 | 権限 |
|-----|------|------|
| **品質監督** | ファクト検証・架空データ禁止ルール強制・ミサキテスト | 差し戻し1回まで |
| **戦略監督** | North Star寄与度スコア付け・Phase分類・9原則チェック | 分類のみ（差し戻しなし） |

### 総エージェント呼び出し

- Round 1（独立点検）: 6 × 4 = 24
- Round 2（相互批判）: 6 × 4 = 24
- 議長統合: 1 × 4 = 4
- 品質監督 + 戦略監督: 2
- **合計: 54エージェント呼び出し**（差し戻し最大4）

## 8. エラーハンドリング

| 問題 | 対応 |
|-----|------|
| ファクト生成失敗（API認証切れ等） | 「データ不足」と明記し続行。推測禁止 |
| エージェントが架空データを出した | 品質監督がブロック → 該当エージェント再実行 |
| パネル間で矛盾する改善案 | 両方残し戦略監督が優先度で判定 |
| 監督差し戻し1回後もPASSしない | メインが手動で該当部分を削除、理由明記 |
| トークンコスト上限到達 | 残ラウンドを議長統合のみに縮退、ログ明記 |
