## 2. 最重要3件（ファネル破壊度順）

### 🔴 #1: C-01 — LP診断5問がLINEに引き継がれず、再度5問聞かれる

**現状**:
- `shindan.js:460-473` で全CTAが `line.me/R/ti/p/@174cxnev` 直リンク化（コメント「LIFFはInstagram WebView でログイン壁」として直リンク化）
- Hero/Sticky/Final CTAも `index.html:1837-1852` で全て同じ直リンク
- LINE Bot `worker.js:3110-3125` の shindan引き継ぎ分岐は存在するが、`source='shindan'` が伝搬しない
- `/api/line-start?source=shindan&session_id=xxx&answers={...}` の受け口(`worker.js:2684`)は生きており **LP側の呼び出しを直リンクに後退させたのが問題**

**インパクト**:
- ミサキが21:30に「5問タップ + LINE登録 + また5問タップ = 合計10問相当」となりブラウザ閉じ
- ファクトパック§3 Lead2 vs LINE登録15 の乖離も、この二度手間で登録後すぐブロックされた結果と仮説

**改善案**:
- shindan.js CTAを `lineUrl('shindan','diagnose')` = `/api/line-start?source=shindan&session_id=${sid}&answers=${encoded}` に戻す
- Hero CTA: `lineUrl('hero','see_jobs')` も `/api/line-start` 経由に戻す
- LIFF廃止理由「Instagram WebView」については、広告Meta WebViewから直リンクline.meも開かない既知問題あり(§7)で、LPの全CTAが共通EP経由の設計に戻すのが健全

**コスト**: 2-3時間 / **依存**: C-09/C-18 と連動

### 🔴 #2: C-05 — welcome Quick Reply「求人を見てみる」1択で温度感低層の入口なし

**現状**: `worker.js:3147-3160` で共通welcome QRが「求人を見てみる」1つだけ。

**インパクト**:
- ファクトパック§13「フェーズB（気づき）: 私の給料普通？」のミサキは求人よりも**相場データ**を見たい
- 1択強制でルートが決まり、温度低層は沈黙ブロック化

**改善案**:
- QR 3択化: (1) 求人を見てみる [see_jobs] / (2) 今の年収が妥当か知りたい [check_salary] / (3) 匿名で相談だけ [consult]
- `check_salary` は `/salary-check/` 既存ページまたは worker.js内 faq_salary へルーティング
- `consult` は `rm_contact_intro` または低プレッシャー nurture_warm へ

**コスト**: 30分 / **依存**: 既存 postback ハンドラ拡張

### 🔴 #3: C-03 + C-04 — Hero第一印象「電話営業」+ welcome長文

**現状**:
- `index.html:1434` の緑CTAが最初の視認要素、直下の hero-note が小さい
- `worker.js:3151` welcome本文が3段落約200文字。ミサキは疲れて読まない

**インパクト**:
- Hero離脱 + welcome読まれずブロック の複合要因でファネル最上流が崩れる
- Meta広告クリック20 → LP閲覧15 → Lead2の大半はここで削れている推定

**改善案**:
- Hero CTA文言: 「1分で求人診断する（電話なし・LINE直結）」
- Hero note フォントサイズ UP + bold化
- welcome第一声: 「こんばんは、ナースロビーです。\nまずはどのエリアで働きたいですか?」＋エリアQR即表示（2メッセ目で感情訴求）

**コスト**: 1時間 / **依存**: なし

## 7. 議長コメント

本パネルは**ファネル中盤（LP→LINE引き継ぎ）に1つの巨大な穴**があると結論した。C-01は**コスト2-3時間に対して期待インパクトが全体の30-50%**と試算でき、他のどの改善よりも優先される。

Phase 1全体は1日で完了可能、Phase 2は2日、Phase 3は週内。ただし**Phase 1の約半分(C-11, C-10)は計測系基盤**で、C-01以下のUX改善の**効果測定に必要な前提**となる。計測基盤と体験改善は並行着手が合理的。

C-14（温度感低層のPDFオファー）はファクトパック§13の**フェーズB層を捕まえる初の具体施策**で、North Star（1名成約）達成の中期の核となる。Phase 2優先。

**最終推奨**: 社長承認後、Phase 1を本日中に実装・デプロイし、明日ファクトで効果検証（shindan_complete率、LINE登録後48h継続率、Lead計測安定性）。
