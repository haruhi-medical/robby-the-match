### 解釈

**良いニュース**:
- aica_turn1 → il_area は 77.8% で、v2想定（60-70%）を上回る。**心理ヒアリング起動 → 条件入力着手は順調**

**悪いニュース**:
- `il_facility_type → il_workstyle` で **2.0% に激減**（98%離脱）
- `aica_turn1 → matching_preview` 全体 **1.18%** = ファネル中盤の絞り込みステップで大量離脱
- v2想定の M2→M3 マッチング閲覧 50-60% は **実測の45倍以上**

### 原因仮説（要検証）

1. **施設種別→勤務形態→急ぎ度のステップ数が多すぎる**: 1問ずつのIL/AICAブリッジに切り替えても、4-5ステップ必須は多い
2. **条件選択肢が網羅すぎ**: 「いずれもない」が選びづらい
3. **Pushメッセージのみで再開できない**: 24h無応答→Push未実装
4. **計測タグの抜け**: `il_workstyle` 到達 146 vs その前段 `il_facility_type` 到達 1,711 → ログ取りこぼしの可能性大
5. **テストユーザー混在**: aica_turn1 4,147件中、QA監査テストが多くを占める（4/29だけで10,647events）→ 実ユーザーのファネル率は別途算出必要

### 推奨アクション

🔴 **F1-A**: KPI再定義
- v2の想定値は「希望」ではなく「実測ベースの目標値（現状から30%改善等）」に書き換え
- 「目標値」と「現状値」を明確に分けて表記

🔴 **F1-B**: 実ユーザーのみのファネル算出
- `phase_transitions.user_hash` でテストユーザーをフィルタ
- AICA監査の user_hash を識別する仕組み（`source='qa_audit'`等）を追加

🟡 **F1-C**: 中盤離脱の根本原因調査
- `il_facility_type` で離脱したユーザーの最終メッセージをサンプリング

### 正味コスト試算（運用フェーズ）

```
月運用コスト（実費 + 機会費用）:
  実費（API+Worker）:      ¥35,000-55,000
  人件費（社長フルコミット相当）:  ¥1,000,000-1,500,000
  ────────────────────────────────────
  合計:           ¥1,035,000-1,555,000

  3人/月成約: ¥900,000-1,500,000
  10人/月成約: ¥3,000,000-5,000,000
```

### Break-even

- **月3-5人成約で正味±0**（機会費用込み）
- **月10人以上で利益化**

🟡 **F2-A**: ROI再計算ブロックを以下に置換

```diff

+月コスト（実費）: ¥35,000-55,000
```
