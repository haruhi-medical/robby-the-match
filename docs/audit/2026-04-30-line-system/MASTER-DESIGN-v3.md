## 1. ビジョンと5原則

### Vision
> **看護師1人が応募〜入職〜定着までAIが完全伴走。**
> **人間が触るのは「重要事項説明」「契約・請求」「緊急時」だけ。**
>
> 参照モデル: スターリンク カスタマーサービス（AI完走 / 契約事項のみ人間）

### 戦略決定（2026-04-30 社長確定）

| 区間 | 担当 | 根拠 |
|---|---|---|
| 友だち追加〜応募意思表明・書類・面接対策・条件交渉・退職交渉・定着フォロー | **AI完走** | 社長休日3名ロストの実体験。AIなら24/365で漏らさない |
| **重要事項説明（職業安定法上の義務）** | **人間** | 紹介事業者の法的義務 |
| **契約（病院との契約・請求）** | **人間** | 商流・法務 |
| 緊急時（自殺示唆等14語） | **人間（即対応）** | リスク管理 |

### 5つの設計原則（v2から継承＋更新）

1. **AI-First, Human-Last**: 人間は契約・請求と緊急時のみ。書類・面接・退職交渉・定着まで全部AI
2. **キャラを消した中立AI**: 「AIだから話せる」現象を利用。人間相手では蓋を閉める本音を引き出す装置
3. **No-Type-First**: タップ＞音声＞テキスト。1問ずつしか聞かない（大手のような尋問を避ける）
4. **Just-in-Time Information**: 必要になった時に必要な情報だけ聞く。事前に履歴書フォームを書かせない
5. **Data-Backed Replies**: D1 施設DB（**89,674件**）+ 求人DB（**3,438件**・2026-04-30実測）+ ハローワーク差分を参照して具体数字で答える

### 人間に通知が飛ぶ条件（3ケースのみ・2026-04-30 圧縮）

1. **緊急ワード検出**: 12語（worker.js:291 AICA_EMERGENCY_KEYWORDS が現行実装の正） → 即時Slack通知＋AICA停止

## 4. 心理ヒアリング 4ターンの設計（AICA中核）

### 思想
- キャラを消した中立AI = 「AIだから話せる」を最大化
- 1ターン1質問、共感は最初の1回のみ
- 200文字以内、絵文字なし、呼称は「○○さん」（「様」NG）

### 7軸自動分類

ユーザーの最初の発言を heuristic + LLM で7軸に分類:

| 軸 | キーワード例 | Turn 2 質問テンプレ |
|---|---|---|
| **time**（労働時間） | 夜勤/残業/休み/シフト | その負担は全員に共通ですか、それとも偏っていますか？ |
| **relationship**（人間関係） | 師長/同僚/医師/陰口 | その関係は特定の方との間だけですか、それとも部署全体の空気ですか？ |
| **salary**（給与） | 給料/年収/手当 | 同世代の方と比べて、評価や役割に納得感はありますか？ |
| **career**（キャリア） | 成長/認定/専門/スキル | 今の業務の中で、伸びていないと感じるのはどんな時ですか？ |
| **family**（家庭） | 育児/介護/結婚/妊娠 | 両立のために『これさえあれば』という条件はありますか？ |
| **vague**（漠然） | なんとなく/疲れた/泣いた | 強いて選ぶなら、それは『人』に関することか『働き方』に関することか |
| **emergency**（緊急） | 死にたい/パワハラ等14語 | **即停止＋人間通知** |

### Turn構成

```
Welcome（自動送信）
  「最大4つの質問で『本当に必要な条件』を整理した後、
    具体的な求人をご提案します。
    今、お仕事で気になっていることを一言で？」
        ↓
TURN 1: 表層の言語化（共感1回OK）
        ↓ 軸自動分類
TURN 2: 環境・構造の特定（軸別質問）
        ↓
TURN 3: 具体エピソード・本音
        「直近で『もう限界だ』と感じた場面は？」
        ↓
TURN 4: 理想像・譲れない条件
        「もし環境が整っていれば『続けたい』と感じる条件は？」
        ↓
SUMMARY: 「○○さんに必要なのは『（抽出条件）』を整えること」
        → CONDITION_HEARING へ自動遷移（沈黙しない）
```

### IL→AICA 共感ブリッジ（昨日実装、4/29）

**問題**: 旧構造ではILフロー（条件カルテのボタン選択）中に「もう夜勤がきつくて限界です…」と打ち込むと「もう一度お選びください👇」と冷たく返してた。

**解決**: ILフェーズで以下を検出したらAICA 4ターンに自動転回:
- 10文字以上の長文
- かつ感情キーワード（辛い/限界/泣/疲弊/辞めたい/不安/ストレス/病んで等）
- または看護現場語彙（夜勤/プリセプター/師長/二交代/三交代等）+ 15文字以上

**コード**: `api/worker.js` `isEmotionalVentingText()` (584行) + IL phaseでフォールスルー (11200行)

## 7. BOT/人間の境界制御

### Silence-As-Default

BOTが応答すべきか迷う時は **黙る**。誤発火が最大の離脱要因。

```
phase = HANDOFF (人間対応中)
  → BOT沈黙。Slackで人間が !reply で返信
phase = PAUSED (一時停止)
  → BOT沈黙。ユーザーが「再開」を押すまで待つ
phase = EMERGENCY
  → BOT沈黙＋Slack緊急通知
```

### 自動転回ルール

| 検出 | 転回先 |
|---|---|
| ILフェーズ中の感情テキスト | AICA TURN1（昨日実装） |
| MATCHING/JOB_QA中の沈黙60秒 | AI相談誘導 Quick Reply |
| 「人と話したい」明示 | HANDOFF（Slack通知） |
| 緊急12語検出（実装現行値） | EMERGENCY（即Slack通知＋BOT沈黙） |
