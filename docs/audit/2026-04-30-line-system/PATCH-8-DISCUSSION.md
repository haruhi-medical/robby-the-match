## 議論項目

### ⑤ 経験分野の入力方法（複数キャリア対応）

#### 問題
- 看護師は **複数科を経験** しているのが普通
  - 例: 「循環器急性期病棟3年、呼吸器内科慢性期病棟5年」
- 単一選択（「内科系」をタップ）だと **「全部の科を経験あり」** と誤解される

#### Patch 8 で取った対応（暫定）
1. QRの最初に **「自分で書く（推奨）」** を配置
2. それ以下の選択肢は「**主に**内科系」「**主に**外科系」と「主に」を付加
3. AIプロンプトに具体例を入れて促す:
   > 「これまでのご経験を、ざっくりで構いませんのでお書きください。
   > 例: 『循環器急性期で3年、その後呼吸器内科で5年』のような形」
4. AIが自由テキストを構造化（科 + 期 + 年数）して entry.fields_experienced に保存

#### 残課題
- 自由記述だとAIの構造化精度が未知数
- 構造化に失敗した場合のリカバリ（「もう少し詳しく」追加質問）
- 入力負荷の高さ（書きたくない人が離脱しないか）

#### 次の打ち手案
A. **構造化UI** を別途作る: LIFF or 独自フォームで「科 + 病棟タイプ + 年数」を3行入力可
B. **AI追加質問** で補完: 「ICU8年」と書かれたら「もう少し詳しく、ICUの中でも何科でしたか？」
C. **QRで多段選択**: 1つ選んだら「他にも経験ありますか？」をQRで問う
D. **音声入力推奨**: 「マイクボタンで話してくださいね」を強調 → AIが構造化

→ **A は工数大、B は最小実装、C は途中で疲れる、D は AIキャラ思想に合致**

#### Claude Code の推奨
**B + D の組み合わせ**:
- AIプロンプトに「もう少し詳しく聞く」追加質問テンプレを実装（半日）
- Welcome 文言に「マイクで話す方が楽です」を強調（5分）

### ⑥ 選択肢 vs パーソナライズの本質的トレードオフ

#### 社長指摘
> 選択肢がしっかりしていると、自分で入力しないからパーソナライズされないよね。

これは本質的な設計トレードオフ。

#### 状況整理

| アプローチ | 入力負荷 | パーソナライズ精度 | 離脱リスク |
|---|---|---|---|
| 選択肢のみ | 低 | 低（典型パターンに分類） | 低 |
| 自由記述のみ | 高 | 高（個別性を捉える） | **高**（書きたくない人離脱） |
| 選択肢 + 自由記述 | 中 | 中 | 中 |
| **AI対話（口頭で深掘り）** | 中 | **高**（AIが個別性を引き出す） | 低（音声入力なら更に低） |

#### AICA設計が目指すべき方向

**AICA 4ターン心理ヒアリング**は本来、選択肢ではなく「対話で本音を引き出す」装置。
「選択肢で終わらせる」発想自体が AICA 設計と矛盾する。

しかし条件カルテ（CONDITION_HEARING）は事実情報の収集で、対話より構造化が必要。
ここに**緊張がある**。

#### 提案する設計原則

```
┌──────────────────────────────────────────────────┐
│ Phase 1: 心理ヒアリング (TURN 1-4)                │
│   選択肢ゼロ。完全自由記述で本音を引き出す       │
│   AI軸自動分類で個別性を把握                     │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│ Phase 2: 条件カルテ (CONDITION_HEARING)           │
│   ハイブリッド方針:                              │
│   - 数値項目（年数・夜勤回数・給与）→ 数字入力   │
│   - 構造化困難項目（経験分野）→ 自由記述+AI構造化│
│   - 単純選択項目（働き方・施設タイプ）→ QR      │
│   全項目に「自分で書く」を**先頭**に配置         │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│ Phase 3: マッチング後の質問・問い合わせ           │
│   完全自由記述 + AI応答（既存JOB_QA）            │
│   選択肢なし。本人の言葉で質問してもらう         │
└──────────────────────────────────────────────────┘
```

#### Patch 8 で実施した方針との整合

✅ Welcome QR: 「自分の言葉で書く」を先頭に → 設計原則と整合
✅ 経験年数: QR廃止・数字フリー入力 → 数値項目は数字で
✅ 経験分野: 「自分で書く（推奨）」先頭 + 補助選択 → ハイブリッド

#### さらに考えるべきこと

1. **音声入力の活用**: 看護師は仕事中はキーボード打てない。音声 = 個別性引き出し最強
2. **AI追加質問の質**: 「もう少し詳しく」「具体的には？」を文脈に応じて出せるか
3. **QRと自由入力の心理学**: QRの「楽さ」がパーソナライズを殺す → QRラベルを「ざっくり選ぶ」と表現する等の工夫
4. **離脱許容**: 自由記述で離脱するユーザーは、そもそも本気度が低い説 → 雑にQRで取るより離脱して良い
