## 0. エグゼクティブサマリー（社長が1分で見切る）

**結論**: 入職まで20フェーズ。うち **12フェーズ ✅ 実装済** / **5フェーズ 🟡 部分実装（P0バグ or 欠落要素あり）** / **3フェーズ 🔴 未実装**。

**実装済の強力な部分**:
- 広告→LP→LINEの引き継ぎ（session_id 永続化 + LIFF + utm継承）
- アキネーター型UI（エリア→施設タイプ→働き方→温度感）
- D1 24,488施設 + 求人2,936件からのマッチング
- Flexカルーセル5枚表示
- AI多層フォールバック（OpenAI→Claude→Gemini→Workers AI）
- 緊急キーワード検出（14語→ホットライン）
- 自由テキストNLP（47都道府県検出）
- 電話確認フロー + handoff → Slack → !reply
- 有料職業紹介事業許可 23-ユ-302928

**🟡 部分実装 — 今週潰せる P0**:
1. 匿名プロフィールに「資格・経験年数・転職理由」が載らない → 病院への打診が **中身スカスカ**
2. AI相談の8ターン超で **無限ループ**（consult=extendが機能しない）
3. OpenAI失敗時の Push API 失敗で **完全無応答** の可能性
4. 広告→LP で「30秒診断」↔「1分診断」の時間表記不整合
5. 辞退時の引き止めスクリプトなし → ミサキがSlack経由で人間に詰められる可能性

**🔴 未実装 — Phase 1-2で新規に必要**:
1. **AI書類生成**: 履歴書・職務経歴書・志望動機を最短1分（doda実証で通過率+3pt）
2. **AI面接対策**: 応募施設ごとの想定Q&A + 模擬面接
3. **入職後5回Push**: 当日/数日後/1週/1ヵ月/3ヵ月の自動リマインド

**最も危険な離脱ポイント**:
- **フェーズ5（アキネーター型UI）→ フェーズ6（マッチング）の間**: 「経験年数・転職理由を聞かれないまま求人出された」→ミサキが「え、これだけで？」と不信感、もしくは打診先の病院がミサキを特定できず **打診が不発**
- **フェーズ17（退職交渉）**: 師長の引き止めにミサキが屈する確率が高い（罪悪感70%）→ スクリプトとLINE人的応答ルートの強化が必須

**所要時間の現実**:
- LINE登録まで: **5分**（広告タップ→診断→LINE友だち追加）
- 最初の求人提示: **10分**（LINE登録後に Flex カルーセル）
- 匿名打診完了: **20分**（同日内、individual情報入力後）
- 内定: **平均3週間**（打診→面接調整→面接→内定）
- 入職: **2ヶ月**（内定→退職交渉→入職日）
- 定着確認: **入職後3ヶ月まで**

### 裏で動くシステム

### 実装状況: ✅（2026-04-17 修正済）

- ⚠️ 「いつでもブロックOK」がFVにない（FinalCTAのみ）

### 離脱リスク

- **ローディング速度**: Lighthouse で LCP < 2.5s が必須。FVの画像焼き込みテキストが重いと読む前にスワイプバック

### 今できる改善

## フェーズ2: LP診断5問（Day 0 / 21:47-21:51 / 所要1-2分）

### ミサキの行動
CTA「1分で診断する」をタップ → shindan.html に遷移 → 5問を順にタップ

### 画面遷移（5問）
```
Q1: 働きたいエリアは？
   [東京都][神奈川県][千葉県][埼玉県][その他]
   → ミサキ: [神奈川県]

Q2: エリアを絞り込み
   [横浜・川崎][湘南・鎌倉][相模原・県央][横須賀・三浦][小田原・県西][どこでもOK]
   → ミサキ: [横浜・川崎]

Q3: どんな職場が気になりますか？
   [急性期病院][回復期病院][慢性期病院][クリニック][訪問看護][介護施設][こだわりなし]
   → ミサキ: [クリニック]（急性期に疲れた）

Q4: 希望の働き方は？
   クリニック選択時は [常勤（日勤）][パート・非常勤] の2択
   → ミサキ: [常勤（日勤）]

Q5: 今の転職への気持ちは？
   [すぐにでも転職したい][いい求人があれば][まずは情報収集]
   → ミサキ: [いい求人があれば]  ← ここが核心。urgency=good
```

### ミサキの内心（各問の蓋の下）
- Q1: 「神奈川一択。横浜に住んでる」
- Q2: 「通勤考えたら横浜・川崎」
- Q3: **「…クリニック。夜勤ない方が…いや、でも収入減るのが…」**（3秒悩む）→「いや、クリニック選んでみる」
  - ここで初めて「夜勤をやめたい」本音が蓋の隙間から漏れる
- Q4: 「常勤は譲れない。パートは給料が」
- Q5: **「すぐにでも」は嘘になる。「情報収集」は自分に嘘をついてる気がする。「いい求人があれば」が一番しっくり」**
  - ← ミサキの「辞めたいわけじゃない。でもこのままは嫌だ」がこの選択肢に集約される

### 結果画面
```
診断結果 ─ 横浜・川崎 × クリニック × 常勤 × いい求人があれば

あなたに合いそうな求人は 27件 見つかりました

[LINEで結果を受け取る（無料）] ← CTA
完全無料・電話なし・いつでもブロックOK

📋 引き継ぎコード: ABC123  ← 6文字
```

### 裏で動くシステム
- shindan.js で回答を保持（`A = { pref, area, ft, ws, urg }`）
- `fetch(WORKER_BASE + '/api/shindan-bridge', { method: 'POST', body: { session_id, answers }})` でサーバーに送信
- Worker側で6文字引き継ぎコード生成（例: ABC123）→ Cloudflare KV に30分TTLで保存
- 画面に「LINEで結果を受け取る」CTA表示
- CTA は LIFF URL: `https://liff.line.me/2009683996-7pCYfOP7?sid=XXX` に誘導

### 実装状況: ✅
- shindan.js v6.0、5問構成、施設タイプに応じた働き方分岐
- session_id 引き継ぎ（commit 3b5c599）
- LIFF ID 2009683996-7pCYfOP7 で既友だち対応

### ミサキテスト: 未測定（この部分の単独採点なし）

### 離脱リスク
- **5問の離脱率**: 業界平均は1問あたり10-15%離脱。5問完走で **50-55%残る** 想定
- **Q5の心理負担**: 「すぐにでも」と「情報収集」の間で迷う層が多い。「いい求人があれば」が正解だとわからないと離脱
- **施設タイプの選択肢**: 「こだわりなし」が用意されているのは良いが、看護師以外の施設（言語聴覚士等）が混ざると不信感

### 今できる改善
- Q5の「いい求人があれば」を **デフォルト選択** にする（UX: ラジオボタンの初期値）
- Q3の「こだわりなし」選択時に、後続UIで勝手に決めない（ユーザー意思尊重）
- 進捗バー表示（「あと3問」等）で完走率向上
