## 7. Agent 6の_isClinicバグ: worker.jsコード照合

### FT6-025の報告内容
> `cond_change=facility`の処理（L5281-5286）で`delete entry.facilityType; delete entry.hospitalSubType; delete entry.department;`のみ。`_isClinic`は削除されない。

### worker.jsの実際のコード（L5281-5298）

```javascript
} else if (val === "facility") {
  delete entry.facilityType;
  delete entry.hospitalSubType;
  delete entry.department;
  nextPhase = "il_facility_type";
} else if (val === "workstyle") {
  delete entry.workStyle;
  delete entry._isClinic;
  nextPhase = "il_workstyle";
} else {
  // 全リセット
  delete entry.area; delete entry.areaLabel; delete entry.prefecture;
  delete entry.facilityType; delete entry.hospitalSubType; delete entry.department;
  delete entry.workStyle; delete entry.urgency; delete entry._isClinic;
  nextPhase = "il_area";
}
```

### 照合結果

**Agent 6の報告は100%正確。**

- `cond_change=facility`: `_isClinic`を削除しない（L5281-5286） -- バグ確認
- `cond_change=workstyle`: `_isClinic`を削除する（L5290） -- 正常
- `cond_change=all`: `_isClinic`を削除する（L5296） -- 正常

### バグの影響

クリニック選択 → condition_change → 「施設タイプを変える」 → 病院選択 → il_workstyleで2択（常勤/パート）が表示される。
正しくは病院なので4択（日勤/夜勤ありOK/パート/夜勤専従）が表示されるべき。

### 修正方法

L5285に `delete entry._isClinic;` を1行追加するだけで修正可能。

なお、`_isClinic`は`il_ft=clinic`選択時にL5214で設定され、`il_urgency`通過時にL3792で削除される設計。
cond_change=facilityでは`il_urgency`を再通過するため本来は途中で消えるが、
`il_facility_type`で病院を選択した場合、`_isClinic`がtrueのまま`il_workstyle`に到達するため、
**`il_workstyle`到達前の時点で2択が表示されるバグが発生する。**
