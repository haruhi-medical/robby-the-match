## 1. 給与表示の品質

### 結果: 不合格 -- 重大バグあり

**致命的問題: `月給0.2万円（※短時間勤務の可能性）` が48件存在**

| salary_display | 件数 |
|---|---|
| `月給0.2〜0.2万円（※短時間勤務の可能性）` | 27 |
| `月給0.2万円（※短時間勤務の可能性）` | 21 |

**原因**: `hellowork_rank.py` の `parse_salary()` で、`salary_form` が空文字 or 「月給」「時給」以外の値の場合、時給（例: 2,050円）が月給として処理される。`2050 / 10000 = 0.2万円` と表示されてしまう。

**実例**:
| employer | salary_min | salary_max | 表示 |
|---|---|---|---|
| 町田市民病院 | 2,050 | 2,050 | 月給0.2万円 |
| 山之内病院（夜勤専従） | 2,413 | 2,413 | 月給0.2万円 |
| 立川市役所（保育園看護師） | 1,707 | 1,707 | 月給0.2万円 |

**修正案**: `parse_salary()` に以下のガード追加:
```python
# salary_formに関係なく、金額が10,000円未満なら時給と判定
if not is_hourly and low_num > 0 and low_num < 10000:
    is_hourly = True
```

### 正常パターン（上位表示）

|---|---|
| 時給2,000円 | 81 |
| 時給1,700〜1,800円 | 66 |

正常パターンの幅表示（`〜`区切り）は問題なし。

## 3. 同一事業所の重複占有

### 結果: 要注意 -- 重複が深刻

| employer | 件数 |
|---|---|
| 株式会社 創生事業団 首都圏事業部 | **77** |
| 株式会社 ミレニア | **49** |
| 株式会社ソラスト介護事業本部 | **48** |
| 株式会社三英堂商事 | **44** |
| 株式会社 ASCare 東京支店 | **44** |

`generateLineMatching()` のSQLは `ORDER BY score DESC LIMIT 5 OFFSET ?` のみで、**同一事業所の重複排除（dedup）ロジックがない**。

例: 横浜エリアで検索すると、ソラスト48件中横浜分が上位を占有する可能性が高い。
ユーザーに同じ会社の求人が5件中3件表示されると信頼感が下がる。

**修正案**: SQLに `GROUP BY employer` を入れるか、アプリ層で1事業所最大2件に制限。

## 5. Dランク求人の内容

### 結果: 要注意 -- 表示品質に問題あり

ランク分布:
| rank | 件数 |
|---|---|
| D | **907** (30.9%) |
| B | 860 (29.3%) |
| C | 720 (24.5%) |
| A | 432 (14.7%) |
| S | 17 (0.6%) |

**問題点**: `generateLineMatching()` のSQLにランクフィルタがない。
`ORDER BY score DESC` で下位になりやすいが、求人数が少ないエリア（海老名11件等）ではDランクが表示される可能性が高い。

**Dランク求人の例**:
- `月給0.2万円（※短時間勤務の可能性）` の求人（上記バグ求人）
- 「★希少な夜勤専従看護師 ★好待遇★」— 釣りタイトル
- 東京都の求人が神奈川ナース転職に混入（東日暮里、江戸川区等）

**修正案**: `WHERE rank != 'D'` をSQLに追加。または `WHERE rank IN ('S','A','B')` で上位のみ表示。
