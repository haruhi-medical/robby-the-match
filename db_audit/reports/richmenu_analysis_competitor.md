## 1. レバウェル看護（旧 看護のお仕事）

### LINE公式アカウント概要
- LINE公式: https://page.line.me/583uudxn （情報配信用）
- 友だち数: 業界最大級（年間利用者数トップクラス）

### リッチメニュー構成（推定）
- 求人検索への導線
- キャリアアドバイザー相談
- 転職ノウハウ / コラム
- マイページ / 登録情報

### 登録後のフロー
1. 友だち追加
2. 見たい求人条件を入力（アンケート形式）
3. 条件に沿った求人がLINEで届く（週1回程度）
4. **初回は電話ヒアリングが必須**（10分程度、日時指定可）
5. 以後の連絡はLINE / メール / 電話から選択可能

### 求人検索の仕組み
- LINE上で直接検索する機能はなし
- キャリアアドバイザーが条件に合った求人をLINEメッセージ（URL付き）で送付
- 応募意思やスケジュール調整もLINE返信で完結

### 差別化ポイント
- **「しつこくない」設計**: 求人通知は週1回程度、スワイプで削除可
- 電話→LINE移行がスムーズ（初回だけ電話、以後LINE）
- ユーザー満足度調査No.1（PR TIMES発表）

### ユーザー口コミ
- 「LINEだと気軽で負担が少ない」
- 「興味ない求人はスワイプで消せるのが良い」
- 初回電話は必須なため「完全電話なし」ではない点に不満の声あり

- キャリアパートナー連絡

2. 登録後すぐにおすすめ求人がメール/LINEで届く

4. LINE / メール / 電話でやり取り継続

- キャリアパートナーが条件に合う求人をLINE経由で紹介
- 自分で検索→応募はWebサイト側で行う
- LINEはコミュニケーションツールとしての活用が主

### 差別化ポイント
- **医療従事者によるマッチング**（臨床現場への理解が深い）
- 電話・LINE・メールの使い分けが柔軟
- 登録直後のレスポンスが速い

### ユーザー口コミ
- 「登録してすぐにおすすめの求人を送ってくれた」
- 「LINEと電話をうまく使い分けて利用できた」
- 「やりとりがスムーズで負担が少ない」
- 一方「グイグイ来る」との声も（担当者による）

### ユーザー口コミ
- 「最初のLINE相談がスキマ時間にできて気軽」

## 5. ジョブメドレー

### LINE公式アカウント概要
- LINE公式: https://page.line.me/rhx9852z
- 日本最大級の医療介護求人サイト

### リッチメニュー構成
- 現時点ではリッチメニューは最小限（今後拡充予定と公言）
- LINEログインによるID連携が設計の核

### 登録後のフロー
1. **LINEログインで新規会員登録**（ID連携が同時完了）
2. 登録した経歴・希望条件に沿った求人がLINEで届く
3. 事業所からのスカウト通知もLINEで受信
4. 「LINEで相談」でキャリアサポートへ問い合わせ

### 求人検索の仕組み
- **自分で検索→自分で応募**が基本（セルフサーブ型）
- スカウト機能: 事業所から直接オファーが届く（内定率1.7倍）
- LINEはあくまで通知チャネル（求人レコメンド+スカウト通知）

### 差別化ポイント
- **完全パーソナライズ配信**: 一斉配信を廃止、条件に合う情報のみ送付
- **LINEログイン→ID連携の一気通貫フロー**: 登録時の離脱を最小化
- エージェント介在なし（セルフサーブ型）→ 電話なし
- スカウト経由の内定率1.7倍という実績

### ユーザー口コミ
- 「自分のペースで探せる」
- 「エージェントからの電話がなくて楽」
- スカウトが多く届くとの声あり

## 7. LINE公式ドキュメント リッチメニュー ベストプラクティス

### サイズ・テンプレート
| 種別 | 推奨サイズ | テンプレート数 |
|------|-----------|--------------|
| 大サイズ | 2500px x 1686px | 7種（6/4/3分割等） |
| 小サイズ | 2500px x 843px | 5種（3/2分割等） |
| ファイル形式 | JPEG or PNG | 1MB以下 |

**推奨: 大サイズ**（情報量が多く、反応率が高い）

### 設定のコツ（LINE公式推奨）
1. **デフォルト表示ON** — トーク画面を開いた時点でメニューが見える状態に
2. **期間指定表示** — キャンペーン時に特定メニューを期間限定表示
3. **自動応答との連携** — テキストアクションで営業時間等の自動回答
4. **クーポン・ショップカード連動** — 利用促進につなげる

### デザイン原則
1. **シンプル設計** — 必要最低限の情報に絞り、十分なスペース確保
2. **高コントラスト配色** — ボタンが目立つ配色
3. **明確なCTA** — 「求人を探す」「相談する」等、短く動詞形で
4. **視線誘導** — 重要情報は左上→右下の自然な流れに配置
5. **アイコン活用** — テキストだけでなくアイコンで直感的に理解

## 25の知見まとめ

### A. ナースロビーが参考にすべき点（10項目）

1. **タウンワーク方式の「条件ボタン直置き」**: 「日勤のみ」「夜勤専従」「高給与」「託児所あり」など看護師の検索ニーズをボタン化すれば、応募率79%増のタウンワーク効果が見込める

2. **ジョブメドレーの「パーソナライズ配信」**: 一斉配信は絶対にやらない。登録条件に合う求人だけを送ることで、ブロック率を最低限に抑える

3. **看護rooの「タブ切り替えリッチメニュー」**: 情報量が多い場合、タブで整理することでユーザーが必要な情報に辿り着きやすい。ただしMessaging API必要

4. **dodaの「診断コンテンツ」をリッチメニューに配置**: ナースロビーの給与診断やエリア相場診断をリッチメニューのボタンに。タップ1回で診断開始

5. **dodaキャンパスの「ステータス別メニュー出し分け」**: 未登録→登録済み→求人紹介中→内定、でリッチメニューを自動切り替え。各段階で最適な導線を提供

6. **レバウェル看護の「週1回配信」ペース**: しつこくない頻度が好評。ナースロビーも求人通知は週1-2回に抑える

7. **ジョブメドレーの「LINEログイン→ID連携一気通貫」**: 登録時の離脱を最小化する設計。LINEで友だち追加→そのまま会員情報入力が理想

8. **マイナビ看護師の「チャットで面談予約」**: テキストベースで面談日程を確定できる機能。電話が苦手な看護師（ペルソナ・ミサキ）に刺さる

9. **セブンイレブンの「今週の新着」**: 定期訪問を促す仕組み。「今週の新着求人」をリッチメニューに

10. **資生堂の「!」アイコン**: タブに通知バッジ風のアイコンを付けて、タップ率向上

### B. ナースロビーが差別化できる点（8項目）

11. **手数料10%の明示**: 大手は紹介手数料30-35%。ナースロビーの10%は圧倒的差別化ポイント。リッチメニューに「手数料10%の理由」ボタンを配置

12. **AIチャットBot「ロビー」**: 大手はほぼ全てが人間アドバイザー。AI即レス+人間フォローのハイブリッドは業界唯一

13. **電話なし完全LINE完結**: 大手5社は全て初回電話ヒアリングが必須。ナースロビーは最初からLINEのみで完結可能に（→ 最大の差別化）

14. **ハローワーク求人の統合表示**: 大手は非公開求人中心。ナースロビーはハローワークAPI連携で公開求人も含めた網羅的な情報提供が可能

15. **神奈川県特化の深い情報**: 大手は全国対応。ナースロビーは神奈川212施設の詳細DB（病床数・看護体制・離職率等）で深さで勝負

16. **地域コミュニティ感**: 大手は事務的。ロビーのキャラクター性とカジュアルなトーンで「相談しやすさ」を演出

17. **即レス**: 大手は担当者が複数求職者を抱えるため返信遅延が常態化。AIは即レス可能

18. **診断→求人→相談の一気通貫LINE体験**: 大手はLINEをあくまで「通知ツール」として使用。ナースロビーはLINE内で診断から求人閲覧、相談まで完結

### C. リッチメニューに入れるべき機能（4項目）

19. **「求人を探す」ボタン**（最上部・最大面積）: タウンワーク事例から、最も使われるボタンを最上部に。タップ→エリア選択or条件選択→求人一覧表示

20. **「給与診断」ボタン**: dodaの診断コンテンツ戦略。ナースロビーの強みである給与診断機能への直リンク。タップ1回で開始

21. **「ロビーに相談」ボタン**: AIチャット起動ボタン。他社の「アドバイザーに相談」と差別化し、24時間即レスを訴求

22. **「LINE限定情報」ボタン**: 転職ガイド・面接対策・履歴書テンプレなどコンテンツへの導線。セブンイレブンの「今週の新商品」的に「今週のコラム」で定期訪問促進

### D. リッチメニューに入れるべきでない機能（3項目）

23. **「電話で相談」ボタン**: ナースロビーの差別化は「電話なし」。電話ボタンを置くと大手と同じ土俵に乗ってしまう

24. **「会社概要」「利用規約」**: リッチメニューの貴重なスペースを非アクション系の情報で埋めない。フッターリンクやWebサイトに任せる

25. **過度な分割（7ボタン以上）**: 看護rooは7+ボタンだが、大手の物量があってこそ。ナースロビーは4-6ボタンでシンプルに。迷わせない設計が小規模事業者の武器
