# リッチメニュー4機能 x 25ペルソナ シミュレーション

> 作成日: 2026-04-06
> 目的: LINE リッチメニューの4機能が、多様な看護師ペルソナにどう使われるかを予測し、UI/UX改善の根拠とする

## リッチメニュー4機能

| ラベル | 機能概要 |
|--------|----------|
| **START** | お仕事探しをスタート（5問→求人10件→担当者紹介） |
| **NEW** | 本日の新着求人 |
| **SEARCH** | 施設検索 |
| **SUPPORT** | 履歴書作成（AI） |

---

## 凡例

- **1st**: 最初にタップする機能
- **2nd**: 2番目にタップする機能
- **NEVER**: 絶対に使わない機能
- **WANT**: こういう機能があったら嬉しい（リッチメニューにない機能）

## 1. ミサキ（28歳、病棟5年目、人間関係に疲れ）

| 項目 | 結果 |
|------|------|
| **1st** | **NEW**（本日の新着求人） |
| **2nd** | START（お仕事探しをスタート） |
| **NEVER** | SUPPORT（履歴書作成） |
| **WANT** | 「今の給料って普通？」給与診断。営業されない安心感の明示 |

**理由**: まだ転職を自覚していない。「ちょっと見るだけ」のNEWが心理的ハードル最低。STARTは5問だけなら試すかもしれないが、「担当者紹介」の文字を見た瞬間に警戒する。履歴書は「まだそこまでじゃない」。

## 戦略的示唆

### 1. SUPPORTの位置づけを再考すべき

60%が「絶対使わない」と回答。リッチメニューの4枠のうち1枠を履歴書作成に割くのはもったいない。ただし**ケイコ（ブランク復職）やマリ（急ぎ）のように、SUPPORTが刺さる層は転職意欲が極めて高い=コンバージョン率が高い**ため、単純に削除するのではなく、タップ後の導線で価値を証明する設計が必要。

**改善案**: SUPPORTを「転職サポート」に拡張し、履歴書作成+給与診断+よくある質問をまとめたハブにする。

### 2. STARTの「担当者紹介」がボトルネック

ミサキ・ユウナ・チカなど**潜在層〜情報収集層は「担当者紹介」の文字で離脱する**。5問の診断自体は魅力的だが、ゴールが「担当者紹介」だと心理的ハードルが高い。

**改善案**: STARTのゴールを「あなたに合う求人10件を表示」に留め、担当者紹介は求人詳細ページからのオプションにする。「電話しません」を明記する。

### 3. NEWは「毎日開く理由」になる最強のリテンション機能

NEWを最初にタップする層は「まだ本気じゃないけど見ておきたい」層。**毎日の新着通知が習慣化すれば、潜在層→顕在層への移行装置になる**。

**改善案**: NEWに「あなたの条件に合う新着」のパーソナライズを追加。初回タップ時に希望条件を2問だけ聞く（エリア+勤務形態）。

### 4. SEARCHは「条件が明確な人」の受け皿

SEARCHを最初にタップする層は条件が明確。だが現状の施設検索だけでは**「日勤のみ」「託児所あり」「夜勤専従」などの条件フィルターが不足**している可能性が高い。

**改善案**: SEARCHに人気フィルターのワンタップボタンを設置（日勤のみ/夜勤専従/パート/ブランクOK/託児所あり）。

### 5. 隠れた最大ニーズ: 「電話が来ない安心感」

ユウナ・ミサキ・チカに共通する「電話恐怖」は、ナースロビーの最大の差別化ポイント（手数料10%・電話営業なし）と直結する。**リッチメニューのどこかに「LINEだけで完結。電話しません。」を常時表示すべき**。

## ペルソナ別フェーズマッピング

| フェーズ | ペルソナ | 特徴 | 最適な最初の機能 |
|---------|---------|------|----------------|
| **潜在期**（まだ転職を自覚していない） | ミサキ, チカ, アスカ | 「見るだけ」。営業感NG | NEW |
| **情報収集期**（条件を調べている） | レイ, ノゾミ, アヤ, ユウナ | 給与・条件を比較中 | NEW or SEARCH |
| **検討期**（具体的に探している） | マイ, リナ, ナオミ, ハルカ, アキ, ミキ, エリ, ヨウコ, カオリ, サチコ | 条件明確。自分で探したい | SEARCH |
| **行動期**（転職を決意） | ユカ, カナ, ユミ, サユリ, トモコ, ミホ, マリ | 今すぐマッチングしたい | START |
| **準備期**（応募直前） | ケイコ, マリ | 履歴書が必要 | SUPPORT |

> **重要**: 潜在期の人数が少なく見えるが、実際のLINE登録者の70%以上は潜在期。このシミュレーションは「LINE登録済み」を前提としているため、潜在期が過小評価されている点に注意。

## 結論: リッチメニュー最適化の優先順位

1. **NEW のパーソナライズ化**（潜在層のリテンション=最大のボリュームゾーン）
2. **START から「担当者」の文字を消す**（心理的ハードル除去）
3. **SEARCH のフィルター強化**（条件明確層の満足度向上）
4. **SUPPORT を「転職サポート」ハブに拡張**（履歴書だけでは弱い）
5. **全体に「電話しません」を明示**（ナースロビー最大の差別化）
