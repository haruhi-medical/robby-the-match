## 詳細分析

### チェック1: 日予算¥500の学習フェーズ充足性 — FAIL (High)

**基準:** 日予算は目標CPA × 5倍以上が最低ライン（Meta推奨）。

**現状の問題点:**

このキャンペーンの最終目標はLINE友だち追加（=リード獲得）だが、計測対象はランディングページビューに設定されている。仮にLPビューを「最適化イベント」と見なしても、以下のとおり予算が構造的に不足している。

```
想定CPC（医療/転職/看護師ターゲット、Instagram日本）: ¥80 〜 ¥200
日予算 ¥500 ÷ 平均CPC ¥140 = 1日あたり約3〜4クリック
5日間合計クリック数 = 15〜25クリック

Metaのアルゴリズム学習に必要な最低イベント数: 50件/週
→ 現状見込みは15〜25件 = 学習フェーズ脱出に必要な数の30〜50%しか達成できない
```

**収益構造との照合:**

```
1件成約 = 営業利益 ¥295,000
許容CPA（LINE登録1件あたり）の逆算:
  LINE登録 → ヒアリング率 80%
  ヒアリング → 紹介率 60%
  紹介 → 内定率 80%
  内定 → 入職率 90%
  登録からの成約率 = 0.80 × 0.60 × 0.80 × 0.90 = 34.6%
  許容CPA（LINE登録1件）= ¥295,000 × 34.6% = ¥102,000 が理論上の上限
  現実的な目標CPA（LINE登録）= ¥1,000 〜 ¥3,000（保守的テスト予算）

推奨日予算 = 目標CPA ¥2,000 × 5 = 最低 ¥10,000/日
または最低でも ¥2,000/日（フィールドデータ収集に必要な最低ライン）
```

**判定:** ¥500/日はMetaのアルゴリズムが機能する最低ラインを大幅に下回る。データ収集すら困難な水準。

### チェック2: CBO vs ABO の選択 — WARNING (Medium)

**現状:** 広告セット単位での予算設定（=ABO: Ad Set Budget Optimization）を採用している（日予算¥500を広告セットに設定）。

**分析:**

| 方式 | 推奨シーン | 現状への適合性 |
|------|-----------|---------------|
| CBO（キャンペーン予算最適化） | 複数広告セットがある場合 | 今回は1セットのみなので差異なし |
| ABO（広告セット予算最適化） | テスト段階・セット数1〜2 | 今回は適切 |

1セット内に2本の広告（AD1/AD3）を入れてMetaに自動配分させる設計は、ABO+同一セット内の配信最適化として機能する。現構造では CBO/ABO の選択は実質的に影響しない。

ただし、将来複数の広告セット（例: リターゲティングセット追加）を作る場合は CBO に移行を推奨。

**判定:** 現時点の構造では問題なし。ただしスケール時の注意事項として記録。

### チェック3: 入札戦略（自動 vs 手動）— WARNING (Medium)

**現状:** 最適化対象「ランディングページビュー」で自動入札（Lowest Cost = 最低コスト入札）を暗黙的に採用。

**問題点:**

自動入札（Lowest Cost）はMetaが学習フェーズ中に大量のデータを必要とする。¥500/日の予算では、アルゴリズムが正確なターゲティングを学習する前に5日間が終了してしまう。

```
Lowest Cost の学習フェーズ要件:
  通常: 7日以内に50件の最適化イベント（Meta公式推奨）
  現状見込み: 5日間で15〜25件のLPビュー（不足）

→ 学習フェーズが完了しないまま広告が終了する可能性: 約80%
```

手動入札（Cost Cap / Bid Cap）は学習フェーズをさらに困難にするため、自動入札の選択自体は正しい。ただし予算が学習要件を満たしていないことが根本問題。

**判定:** 入札戦略の選択（自動）は正しいが、予算不足のため機能しない。

### チェック4: 5日間テストの統計的有意性 — FAIL (High)

**現状:** 5日間、AD1 vs AD3 の2本をA/Bテスト。

**統計的要件:**

広告効果の差を95%信頼区間で検出するために必要なサンプル数（各広告あたり）：

```
想定CTR: 0.5〜2.0%
統計的有意差を出すための最低インプレッション数: 各広告 約3,000〜5,000以上
5日間 × ¥500 / 日 = 合計¥2,500

想定CPM（Instagram, 日本, 医療ターゲット）: ¥800 〜 ¥1,500
¥2,500 ÷ CPM ¥1,100（中間値）× 1,000 = 約2,300インプレッション（合計）
2本に分割すると各約1,150インプレッション

→ 統計的有意差の検出に必要なインプレッションの 20〜30% しか確保できない
```

**結論:** 5日間・¥500/日の予算では、AD1とAD3のどちらが優れているかを統計的に判断できない。サンプルサイズが小さすぎて、偶然による差が結論を左右する。

**campaign_guide.md の基準（CTR 1%以上 → 勝ち）を適用する場合:**

獲得クリック数が15〜25件しかなければ、CTRの差が測定誤差の範囲内に収まってしまい、判断の根拠にならない。

### チェック5: キャンペーン目的の適切さ — FAIL (High)

**現状:** 目的「トラフィック」、最適化「ランディングページビュー」。

**実際のビジネス目標:** LINE友だち追加（= リード獲得）。

**問題の構造:**

```
設定されている目的の最適化先:
  「ランディングページビュー」= LPを開く人を最大化
  → Meta は「LPを開きやすい人」に広告を届ける

実際に必要なこと:
  「LINEボタンをクリックしてLINEに登録する人」= コンバージョン
  → Meta は「コンバージョンしやすい人」に広告を届けるべき

ミスマッチの影響:
  - LPを開いても登録しない層にも広告費を消費する
  - Metaのアルゴリズムが「登録者」を学習できない
  - 同じ¥2,500でリード獲得目的にした場合と比べて効率が著しく低下する
```

**推奨変更:**

Meta Pixel が既に設置済みで、LINEボタンクリック時に `fbq('track', 'Lead')` が発火していることが gemini_instruction.md で確認済み。この状態であれば「リード」目的でのキャンペーン設定が可能。

| 現状 | 推奨 |
|------|------|
| 目的: トラフィック | 目的: リード または コンバージョン |
| 最適化: LPビュー | 最適化: Lead イベント（LINEクリック） |
| Metaが学習: LPを開く人 | Metaが学習: LINEに登録する人 |

**ただし重要な留意点:** Leadイベントを最適化対象にする場合、さらに多くの週次コンバージョン数（最低20〜50件/週）が必要になる。予算¥500/日ではリード目的でも学習が成立しない問題は残る。

### チェック9: Learning Phase脱出見込み — FAIL (High)

**Meta広告の学習フェーズ要件:**

```
学習フェーズ（Learning Phase）脱出条件:
  最適化イベントが 7日間で 50件 以上（Meta公式ガイドライン）
  ※ 最適化イベント = 設定した最適化先のイベント（今回は LPビュー）

現状の見込み:
  日予算 ¥500 ÷ 想定CPC ¥140 = 約 3〜4 LPビュー/日
  5日間合計 = 約 15〜20 LPビュー
  7日間換算 = 約 21〜28 LPビュー

→ 学習フェーズ脱出に必要な50件の 42〜56% しか達成できない
→ このキャンペーンは5日間、学習フェーズのまま終了する可能性が非常に高い
```

**学習フェーズ中の広告の特徴:**

- パフォーマンスが不安定（日によって大きくばらつく）
- CPCが高く、配信効率が悪い
- 取得したデータが信頼性の低いノイズである可能性が高い
- 広告クリエイティブの真の実力が測れない

**必要な日予算の逆算:**

```
目標: 7日間で50件のLPビュー
必要なLPビュー/日: 50 ÷ 7 = 約7件/日
想定CPC: ¥140（中間値）
必要な日予算 = 7 × ¥140 = ¥980/日
推奨日予算（安全マージン込み）= ¥1,500 〜 ¥2,000/日
```

## キャンペーン構造の問題点（構造的欠陥）

### 問題1: 1セット内2広告の真のA/Bテスト不成立

```
現状: 1つの広告セット内にAD1とAD3を両方入れる
      → Meta が両者を自動配分（≠ 統制されたA/Bテスト）

真のA/Bテスト（Meta実験ツール使用）との違い:
  自動配分     | 過去の反応が良い広告に自動的に多く予算を配分する（早期収束バイアス）
  実験ツール   | 50/50で均等に配信、統計的有意差を自動計算

推奨: 予算が増えたら Meta の「A/Bテスト（実験）」機能を使う
      現状の予算では判定不可なので、どちらの方式でも結果は同じ
```

### 問題2: コンバージョンの連鎖が断絶している

```
広告 → LP表示（計測される）→ LINEクリック（Leadイベント発火）→ LINE登録
         ↑                        ↑
      学習対象                 本当に欲しいもの

LPビューを最適化しても、LP→LINEクリックの転換率が低ければ無意味。
現時点でLP→LINE登録の転換率が不明。これを把握することが優先。
```

## 最重要コメンタリー

今回のキャンペーンの設計は正しい方向を向いているが、予算が構造的に不足しており、5日間の結果から意思決定できるデータが得られない可能性が高い。

最も重要な優先順位は次のとおりだ:

1. **このテストの目的を「学習」と割り切る。** ¥2,500でMetaの仕組みを体験し、アカウントに広告配信の実績を積む段階と位置づけよ。

2. **LP → LINE転換率を先に計測せよ。** GA4で「LP到達 → LINEボタンクリック」の転換率を出すこと。もしここが1%以下なら、広告費を増やす前にLP改善が先決だ。

3. **予算を増やす前に目的を変えよ。** 「トラフィック」から「コンバージョン（Lead）」への変更は無料でできる。¥1,500/日でLeadイベントを最適化対象にすれば、Metaが「登録者」を学習し始める。これが最も費用対効果の高い次の一手だ。
